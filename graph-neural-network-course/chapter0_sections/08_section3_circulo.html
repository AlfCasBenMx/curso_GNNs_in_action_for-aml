El Problema

Imagina puntos azules en el centro y puntos rojos alrededor. ¿Puedes separarlos con una línea recta?

❌ Sin Capas Ocultas

¡IMPOSIBLE!
No hay línea recta que los separe

✅ Con 3 Neuronas Ocultas

¡PERFECTO!
La red aprende una frontera curva

🧠 ¿Por Qué Funciona?

Las capas ocultas + activaciones NO lineales permiten que la red aprenda transformaciones complejas. Matemáticamente, esto se debe al Teorema de Aproximación Universal:

"Una red con 1 capa oculta + activación no lineal puede aproximar cualquier función continua, dado suficientes neuronas."

  • Sin capas ocultas (o sin activación): Solo líneas/planos (funciones lineales)
  • Con 1 capa oculta + ReLU: Funciones por partes (círculos, curvas)
  • Con múltiples capas: Patrones jerárquicos complejos

💡 Intuición: Es como "doblar" el espacio de datos. Los puntos azules/rojos que no eran separables en el espacio original, tras pasar por capas ocultas quedan en posiciones donde SÍ se puede trazar un plano divisorio.

¿Por Qué Usamos Estos Ejemplos "de Juguete"?

📋 Código para copiar y pegar: círculo con y sin capas ocultas

A partir de los imports del Paso 0, este ejemplo genera puntos en un círculo, entrena primero un modelo lineal (que no puede aprender bien la frontera curva) y luego una red con capa oculta que sí se adapta al círculo. 

import torch
    import torch.nn as nn

    # Datos en círculo (clasificación binaria)
    def make_circle(n_puntos=400, radio=0.8):
      X = torch.empty(n_puntos, 2).uniform_(-1.2, 1.2)
      radios = X.pow(2).sum(dim=1).sqrt()
      y = (radios < radio).float().unsqueeze(1)
      return X, y

    X, y = make_circle()
    print("Proporción clase 1 (interior del círculo):", y.mean().item())

    # Modelo lineal: 2 -> 1 con Sigmoid (frontera recta)
    modelo_lineal = nn.Sequential(
      nn.Linear(2, 1),
      nn.Sigmoid()
    )

    criterio = nn.BCELoss()
    opt_lineal = torch.optim.SGD(modelo_lineal.parameters(), lr=0.5)

    for epoca in range(1000):
      y_pred = modelo_lineal(X)
      loss = criterio(y_pred, y)

      opt_lineal.zero_grad()
      loss.backward()
      opt_lineal.step()

      if epoca % 200 == 0:
        print(f"[Lineal] Epoch {epoca} | Loss = {loss.item():.4f}")

    with torch.no_grad():
      probs_lin = modelo_lineal(X)
      preds_lin = (probs_lin >= 0.5).float()
      acc_lin = (preds_lin == y).float().mean().item()
    print(f"Exactitud modelo lineal: {acc_lin:.3f}")

    # Modelo con capa oculta: 2 -> 8 -> 1 (frontera curva)
    modelo_profundo = nn.Sequential(
      nn.Linear(2, 8),
      nn.ReLU(),
      nn.Linear(8, 1),
      nn.Sigmoid()
    )

    opt_profundo = torch.optim.SGD(modelo_profundo.parameters(), lr=0.5)

    for epoca in range(1000):
      y_pred = modelo_profundo(X)
      loss = criterio(y_pred, y)

      opt_profundo.zero_grad()
      loss.backward()
      opt_profundo.step()

      if epoca % 200 == 0:
        print(f"[Oculta] Epoch {epoca} | Loss = {loss.item():.4f}")

    with torch.no_grad():
      probs_deep = modelo_profundo(X)
      preds_deep = (probs_deep >= 0.5).float()
      acc_deep = (preds_deep == y).float().mean().item()
    print(f"Exactitud modelo con capa oculta: {acc_deep:.3f}")

🔳 XOR
f*(x) = (x₁>0.5) ⊕ (x₂>0.5)
⭕ Círculo
f*(x) = (x₁² + x₂² < r²)

La red NO conoce estas fórmulas. Solo ve ejemplos (x, y) y trata de descubrir la regla subyacente. En el mundo real, nunca conocemos f* — pero en estos ejemplos podemos verificar si la red la aproximó bien.

Esto es la esencia del aprendizaje supervisado: aproximar una función desconocida a partir de ejemplos.
