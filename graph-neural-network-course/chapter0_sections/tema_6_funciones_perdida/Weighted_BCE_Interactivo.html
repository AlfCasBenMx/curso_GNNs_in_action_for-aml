Weighted BCE Interactivo

Aprende cómo funciona Binary Cross-Entropy con Pesos para Datasets Desbalanceados

Caso de Uso: Detección de Lavado de Dinero (AML)

🎯 ¿Qué es Weighted BCE?

En detección de lavado de dinero (AML), tenemos un problema: 98% de transacciones son legítimas y solo 2% son sospechosas.

Si usamos BCE normal, el modelo aprende a ignorar las transacciones sospechosas (son muy pocas). Weighted BCE soluciona esto penalizando más los errores en la clase minoritaria.

📐 Las 3 Fórmulas que Vamos a Comparar

❌ 1. BCE Sin Peso (Ignora Desbalance)

Fórmula:
L = -(1/n) × Σ[ y·log(p) + (1-y)·log(1-p) ]

📌 Características:
• Entrada: Probabilidades p ∈ [0,1]
• Peso: Todas las muestras pesan igual (w=1)
• Problema: Con 98% legítimas, el modelo solo aprende esa clase
• PyTorch: nn.BCELoss()

✅ 2. BCE Con Peso Manual (Balancea Clases)

Fórmula:
L = -(1/n) × Σ[ w·y·log(p) + (1-y)·log(1-p) ]
donde: w = pos_weight si y=1, w=1 si y=0

📌 Características:
• Entrada: Primero aplicas σ(z) para obtener p
• Peso: Solo amplifica la clase positiva (sospechosas)
• pos_weight recomendado: #Legítimas / #Sospechosas = 49x
• Efecto: Cada error en sospechosa cuesta 49x más
• Implementación: TÚ calculas cada paso manualmente (pedagógico)

⚡ 3. BCEWithLogitsLoss (Producción Optimizada)

Fórmula Interna (Estable):
max_val = max(0, z)
L = pos_weight·y·(max_val - z + log(1 + e^(-|z|))) +
    (1-y)·(max_val + log(1 + e^(-|z|)))

📌 Características:
• Entrada: Logits z directamente (sin sigmoid manual)
• Resultado: Matemáticamente = al método #2
• Ventaja: Log-sum-exp trick evita overflow con valores extremos
• Velocidad: Implementación optimizada en C++/CUDA
• PyTorch: nn.BCEWithLogitsLoss(pos_weight=peso)
• ⭐ Recomendado para producción real

🎯 ¿Cómo Calcular el pos_weight Óptimo?

pos_weight = #Clase_Mayoritaria / #Clase_Minoritaria
Para nuestro caso de AML:

pos_weight = #Legítimas / #Sospechosas pos_weight = 98 / 2 = 49x
📊 Interpretación:
  • pos_weight = 49x significa que cada error en una transacción sospechosa cuesta 49 veces más que un error en una legítima
  • Esto balancea el aprendizaje: aunque hay 49 veces más legítimas, el peso hace que el modelo preste igual atención a ambas clases
  • Con un dataset extremo de 1000 transacciones y 2 sospechosas → pos_weight = 998/2 = 499x
  • En AML real con millones de transacciones, pesos de 100x-1000x son comunes
✅ Regla General:
El pos_weight debe ser igual al ratio de desbalance para que el modelo trate ambas clases con igual importancia durante el entrenamiento.
💡 Conclusión de las Fórmulas:
• Métodos #2 y #3 dan el mismo resultado numérico (diferencia < 10⁻⁶)
• La diferencia entre #2 y #3 es implementación, no matemática
• #3 es mejor en producción: más rápido, estable y conveniente
• #2 es mejor para aprender: ves cada paso claramente

🧮 Cálculo Paso a Paso (Método 2: BCE Con Peso Manual)

📐 Fórmula Completa de Weighted BCE:

L = -(1/n) × Σ[ w·y·log(p) + (1-y)·log(1-p) ]
Donde:
• n = número total de transacciones
• y = etiqueta real (1=sospechosa, 0=legítima)
• p = predicción del modelo (probabilidad de ser sospechosa)
• w = peso para cada muestra:
   - Si y=1 (sospechosa): w = pos_weight (ej: 49x)
   - Si y=0 (legítima): w = 1 (sin amplificación)
🔍 ¿Cómo funciona el peso?
El término w·y hace que el peso solo afecte cuando y=1 (sospechosa):
  • Si y=1: Loss = -pos_weight × log(p) → Amplificado ×49
  • Si y=0: Loss = -log(1-p) → Sin amplificación (el término w·y se anula)
PASO 1: Aplicar Sigmoid a los Logits

Convertir logits (valores crudos del modelo) a probabilidades:

p = σ(z) = 1 / (1 + e^(-z))

Ejemplo: Si z=2.0 → p=σ(2.0) ≈ 0.88 (88% probabilidad de ser sospechosa)

PASO 2: Calcular Pérdida Individual con Peso

Para cada transacción, aplicar la fórmula según su etiqueta:

  • Si sospechosa (y=1):
    Lossi = -pos_weight × log(p) = -49 × log(0.88) ≈ 6.27
    El peso amplifica el error
  • Si legítima (y=0):
    Lossi = -log(1-p) = -log(0.12) ≈ 2.12
    Sin amplificación (peso = 1)
PASO 3: Promediar Todas las Pérdidas

Sumar todas las pérdidas individuales y dividir por n:

Losstotal = (Loss1 + Loss2 + ... + Lossn) / n

Este valor es el que se usa para entrenar el modelo mediante backpropagation

📋 Cálculos Detallados por Transacción:

Fórmula aplicada: Li = -[ w·y·log(p) + (1-y)·log(1-p) ]

📊 ¿Por Qué es Mejor BCE con Peso?

⚠️ Problema del Desbalance:
Con 98 legítimas y 2 sospechosas, un modelo sin peso solo necesita acertar en las legítimas para tener 98% de precisión. ¡Puede ignorar completamente las sospechosas!

🎯 Contribución Individual al Loss Total

Cada barra representa cuánto contribuye una transacción al loss total:

❌ Sin Peso (Ignora Minoritaria)

Todas las transacciones contribuyen por igual → El modelo aprende a predecir solo la clase mayoritaria

✅ Con Peso (Balancea Clases)

Las sospechosas contribuyen 49x más → El modelo DEBE aprender ambas clases

✅ Conclusión Visual:
Con el peso, el modelo aprende que equivocarse en una sospechosa cuesta 49x más que en una legítima. Esto lo fuerza a balancear su atención entre ambas clases, aunque haya muchas más legítimas.

🎯 Resultado: El modelo aprende a detectar la clase minoritaria sin ignorarla por el desbalance.
