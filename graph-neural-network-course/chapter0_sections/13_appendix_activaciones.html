Funciones de Activación: A Fondo

Softmax, ReLU, saturación y el problema del vanishing gradient

Las funciones de activación son el corazón de las redes neuronales. Sin ellas, no importa cuántas capas apiles: el resultado sería equivalente a una única transformación lineal. En esta sección exploraremos en profundidad cada función, sus derivadas, y por qué algunas causan problemas durante el entrenamiento.

🔴 Sigmoid (Función Logística)
σ(x) = 1
1 + e−x
Derivada: σ'(x) = σ(x) · (1 − σ(x))

Características:

  • Comprime cualquier valor a (0, 1)
  • Ideal para probabilidades binarias
  • Suave y diferenciable en todo punto
  • Centrada en 0.5, no en 0
⚠️ El Problema de Saturación

Cuando |x| es grande, σ(x) se aplana:

σ(5) = 0.9933 → σ'(5) = 0.0066
σ(10) = 0.99995 → σ'(10) = 0.00005

Resultado: Los gradientes se vuelven minúsculos → las capas profundas apenas aprenden.

🔵 Tanh (Tangente Hiperbólica)
tanh(x) = ex − e−x
ex + e−x = 2σ(2x) − 1
Derivada: tanh'(x) = 1 − tanh²(x)

Ventajas sobre Sigmoid:

  • Salida centrada en 0 → mejor para capas ocultas
  • Derivada máxima = 1.0 (vs 0.25 de sigmoid)
  • Rango simétrico (-1, 1)
🔄 Sigmoid vs Tanh

Tanh es simplemente una versión reescalada y desplazada de Sigmoid:

tanh(x) = 2 · σ(2x) − 1

# En Python:
tanh = 2 * torch.sigmoid(2*x) - 1

Aún sufre saturación, pero menos severa que sigmoid.

🟢 ReLU (Rectified Linear Unit)
ReLU(x) = max(0, x)
Derivada: ReLU'(x) = 0 si x < 0, sino 1

¿Por qué revolucionó deep learning?

  • Derivada = 1 en zona positiva → sin saturación
  • Computacionalmente muy eficiente
  • Induce sparsity (muchas neuronas en 0)
  • Permite entrenar redes muy profundas
💀 El Problema de "Dead ReLU"

Si una neurona cae en zona negativa y se queda ahí:

Para x < 0: ReLU(x) = 0
Derivada = 0 → Gradiente = 0
→ ¡La neurona nunca se actualiza!

Solución: Leaky ReLU, PReLU, ELU

⚡ Por qué ReLU mitiga el Vanishing Gradient
❌ Con Sigmoid (5 capas)
∂L/∂a₅ × 0.25 × 0.25 × 0.25 × 0.25
Gradiente final: 0.25⁴ = 0.004 (¡casi nada!)
✓ Con ReLU (5 capas)
∂L/∂a₅ × 1 × 1 × 1 × 1
Gradiente final: 1⁴ = 1 (¡se preserva!)
💡 Clave: En backpropagation, los gradientes se multiplican capa por capa. Con sigmoid, cada multiplicación por ~0.25 reduce el gradiente exponencialmente. Con ReLU (en zona positiva), multiplicar por 1 preserva el gradiente.
🟡 Variantes de ReLU
Leaky ReLU
f(x) = max(αx, x)
donde α ≈ 0.01
  • Pendiente pequeña en x < 0
  • Evita dead ReLUs
  • Derivada nunca es exactamente 0
PReLU (Parametric ReLU)
f(x) = max(α·x, x)
α es aprendible
  • El parámetro α se aprende
  • Cada neurona puede tener su propio α
  • Más flexible que Leaky ReLU
ELU (Exponential LU)
f(x) = { x si x ≥ 0
f(x) = { α(eˣ−1) si x < 0
  • Suave en x = 0 (diferenciable)
  • Salidas pueden ser negativas
  • Converge más rápido que ReLU
SELU (Scaled ELU)
f(x) = λ · ELU(x, α)
λ ≈ 1.0507, α ≈ 1.6733
  • Auto-normalizante
  • Mantiene media ≈ 0 y varianza ≈ 1
  • No necesita BatchNorm

🔷 Activaciones Modernas: Softplus y GELU
Softplus
Softplus(x) = ln(1 + ex)

Idea: Una versión "suave" de ReLU

  • Completamente diferenciable (sin esquina)
  • Siempre positiva
  • Se aproxima a ReLU cuando x → ∞
  • Derivada = σ(x) = Sigmoid
GELU (Gaussian Error LU)
GELU(x) = x · Φ(x)
donde Φ es la CDF de N(0,1)

Usada en: GPT, BERT, Transformers

  • Combina propiedades de ReLU y dropout
  • Suave y no-monótona
  • Mejor para NLP y Transformers
  • Aproximación: 0.5x(1 + tanh(...))

🎓
Reglas prácticas:
• Capas ocultas: Empieza con ReLU. Si tienes dead neurons, prueba Leaky ReLU o GELU.
• Clasificación binaria (salida): Sigmoid
• Clasificación multiclase (salida): Softmax
• Transformers/NLP: GELU
• RNNs/LSTMs: Tanh (puertas) + Sigmoid (gates)
