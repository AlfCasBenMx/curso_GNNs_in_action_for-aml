Forward Pass

De entrada a predicción

✅ Al terminar esta sección deberías poder:

  • Describir en palabras el flujo Input → Capas ocultas → Salida (forward pass).
  • Seguir una función forward en código y mapear cada línea a las capas/activaciones del modelo.
  • Razonar sobre las dimensiones: dado un shape de entrada y pesos, deducir el shape de la salida en cada capa.

El forward pass es cuando los datos fluyen hacia adelante desde la entrada hasta la salida, aplicando operaciones matemáticas capa por capa. En cada capa:

  1. Transformación lineal: z = W·x + b
  2. Activación: a = σ(z)
  3. La salida a se convierte en entrada para la siguiente capa
🏭
Analogía: Línea de ensamblaje con dimensiones

Como una fábrica donde el producto pasa por estaciones:
- Entrada: Materia prima (vector de 4 números)
- Capa 1: Procesa y expande a 8 componentes
- Capa 2: Refina a 4 componentes
- Salida: Producto final (ej: 2 clases)

💡 Dimensiones: [batch, 4] → [batch, 8] → [batch, 4] → [batch, 2]

Input x
→
Capa 1
W₁x+b₁ → ReLU
→
Capa 2
W₂h+b₂ → ReLU
→
Output ŷ

🔄 Visualización del Forward Pass

Observa cómo los datos se distribuyen en pulsos desde cada capa hacia la siguiente

💡 Pulso 1 (azul) → Pulso 2 (violeta) → Pulso 3 (naranja) | El ciclo se repite cada 3 segundos 

def forward(self, x):
    x = F.relu(self.layer1(x))   # Capa 1 + ReLU
    x = F.relu(self.layer2(x))   # Capa 2 + ReLU
    x = self.output(x)           # Salida
    return x

La función de pérdida (loss) es un número único que cuantifica qué tan mal está prediciendo la red. Valores más bajos = mejores predicciones. El objetivo del entrenamiento es minimizar este número.

📊 Forward Pass → Cálculo del Loss

Los datos viajan hacia adelante, y al final comparamos predicción vs realidad

Iteración: 1 | Loss actual: —
Historial:

💡 El prediction (ŷ) y el target real (y) llegan al Loss → se calcula el error

🎯
Analogía: Tiro al blanco

El loss mide la distancia al centro:
- Loss = 0: ¡Perfecto! Justo en el blanco
- Loss pequeño (0.1): Muy cerca, buen tiro
- Loss grande (5.0): Lejos del objetivo

La elección de la función de loss depende del problema (regresión vs clasificación).

📐

MSE - Mean Squared Error

La función de pérdida estándar para regresión (predecir valores continuos)

¿Qué mide MSE? El promedio de los errores al cuadrado entre tus predicciones y los valores reales. Mientras más pequeño el MSE, mejor está prediciendo tu modelo. Un MSE de 0 significa predicciones perfectas.

MSE = 1
n Σ (yi − ŷi)²
n
número de muestras
yi
valor real (ground truth)
ŷi
predicción del modelo
🎮 Calculadora Interactiva de MSE (1 muestra)
08.010
05.010
Interpretación:
La predicción está a 3 unidades del valor real.
Error² = (y - ŷ)²
= 9.00
📊 Diagrama Visual: MSE con 3 Muestras
Valor Real (y)
Predicción (ŷ)
Error
Muestra y (real) ŷ (predicción) y - ŷ (y - ŷ)²
1 5 3 5 - 3 = 2 4
2 2 4 2 - 4 = -2 4
3 7 6 7 - 6 = 1 1
Suma de errores² = 4 + 4 + 1 = 9
MSE = Σ(errores²) ÷ n = 9 ÷ 3 = 3.0
📝 Otro Ejemplo: Batch con valores diferentes
Muestra 1
y = 2
ŷ = 1
(2-1)² = 1
Muestra 2
y = 0
ŷ = 1
(0-1)² = 1
Muestra 3
y = 1
ŷ = 2
(1-2)² = 1
Suma de errores²: 1 + 1 + 1 = 3 → MSE = 3/3 = 1.0
⚠️ ¡Importante! Esto es COINCIDENCIA:

Que cada error² sea exactamente 1 y que MSE = 1.0 es pura coincidencia por los valores elegidos. En la práctica, los errores serán valores diferentes (0.23, 1.87, 0.05, etc.) y MSE será cualquier número positivo.

Ejemplo real: y=[3.2, 1.5, 4.0], ŷ=[2.8, 1.9, 3.7]
Errores²: (0.4)²=0.16, (-0.4)²=0.16, (0.3)²=0.09 → MSE = 0.41/3 = 0.137

📏 ¿Cuándo es "bueno" o "malo" el MSE?

El valor de MSE depende de la escala de tus datos. No hay un número mágico que sea "bueno" para todos los problemas.

Prediciendo temperaturas (°C)
MSE = 4 → Error promedio ≈ 2°C
✓ Probablemente aceptable
Prediciendo precios ($1000s)
MSE = 4 → Error promedio ≈ $2000
✗ Podría ser muy malo
Prediciendo probabilidades (0-1)
MSE = 0.01 → Error promedio ≈ 0.1
✓ Generalmente bueno
💡 Tip: RMSE (raíz de MSE) te da el error en las mismas unidades que tus datos. RMSE = √MSE. Si MSE = 9, entonces RMSE = 3 (mismas unidades que y).
💡 ¿Por qué elevar al cuadrado? (3 razones importantes)
1. Hace positivos todos los errores
Sin cuadrado, errores positivos y negativos se cancelarían:
❌ Sin cuadrado
+3 + (-3) = 0
¡Parece perfecto pero no lo es!
✓ Con cuadrado
9 + 9 = 18
Refleja el error real
2. Penaliza más los errores grandes
Esto es deseable: un error de 4 es mucho peor que dos errores de 2.
Error = 1
1² = 1
Error = 2
2² = 4
Error = 3
3² = 9
Error = 4
4² = 16
Error = 5
5² = 25
¡El error crece exponencialmente! Esto fuerza al modelo a evitar errores grandes.
3. Derivada simple para backpropagation
La derivada de x² es 2x, lo cual hace el cálculo de gradientes muy eficiente:
L = (y - ŷ)² → ∂L/∂ŷ = 2(ŷ - y)
Solo multiplicación y resta, ¡muy rápido de calcular!
🔄 Alternativa: MAE (Mean Absolute Error)

Otra opción es usar el valor absoluto en lugar del cuadrado:

MSE
Promedio de (error)²
Penaliza mucho los errores grandes
MAE
Promedio de |error|
Más robusto a outliers

Cross-Entropy Loss

La función de pérdida estándar para clasificación (categorías)

¿Qué mide Cross-Entropy? Qué tan "sorprendida" estaría la red si le dijeras la respuesta correcta. Si la red predice 90% de probabilidad para la clase correcta, la sorpresa (loss) es baja. Si predice solo 10%, la sorpresa es alta → loss alto.

CE = − Σ yi · log(ŷi) simplifica a: −log(ŷclase real)
Solo la clase correcta contribuye al loss (porque las demás tienen y=0)
🎮 Simulador de Cross-Entropy
0.01 (mal) 0.70 0.99 (bien)
Distribución de probabilidades:
C0
C1 ✓
C2
CE = −log(Pcorrecta)
−log(0.70) = 0.36
✓ La red está bastante segura de la clase correcta. ¡Buen trabajo!
📝 Ejemplo: Clasificación con 4 clases
1️⃣ Logits (salida de la red)
z = [2, 1, -1, 0]
2️⃣ Softmax → Probabilidades
[0.59, 0.22, 0.03, 0.16]
3️⃣ CE (clase real = 1)
−log(0.22) = 1.51
📈 Si la red mejora...
P(clase 1) = 0.22 → P(clase 1) = 0.75 | CE = 1.51 → CE = 0.29 ↓81%
📊 Forma de la función −log(p)
p → 0: Loss → ∞ (muy malo) p → 1: Loss → 0 (perfecto)

⚖️

BCEWithLogitsLoss

Para clasificación binaria - más estable numéricamente

BCE = −[y·log(σ(z)) + (1−y)·log(1−σ(z))]
donde z = logit (salida sin sigmoid), σ = sigmoid, y ∈ {0, 1}
✓ ¿Por qué BCEWithLogits?
  • Usa log-sum-exp trick para estabilidad
  • Evita underflow/overflow de sigmoid
  • Combina sigmoid + BCE en una operación
  • Gradientes más precisos
📊 Ejemplos numéricos
z=2.0, y=1: loss = 0.13 (bien)
z=-2.0, y=1: loss = 2.13 (mal)
z=0.0, y=1: loss = 0.69 (incertidumbre)
⚠️ ¿Qué son Underflow y Overflow?

Las computadoras tienen precisión limitada para representar números decimales. Cuando los números se vuelven extremadamente pequeños o grandes, ocurren problemas:

📉 UNDERFLOW
Número demasiado pequeño → se redondea a 0
sigmoid(-100) = ?
= 0.0000000...0000
≈ 0 (¡perdemos precisión!)
Problema: log(0) = -∞ 💥
📈 OVERFLOW
Número demasiado grande → se vuelve infinito
exp(1000) = ?
= 10434
≈ ∞ (¡número imposible!)
Problema: operaciones con ∞ = NaN 💥
🔍 El problema con sigmoid + log:
Paso 1
Logit muy negativo: z = -100
Paso 2
Sigmoid: σ(-100) = 1/(1+e100) ≈ 0.0 (underflow!)
Paso 3
Log: log(0.0) = -∞ 💥 ¡Error!
✓ BCEWithLogits evita esto:
Usa trucos matemáticos (log-sum-exp) para calcular log(sigmoid(z)) de forma estable, sin calcular sigmoid primero. Así evita que números muy pequeños o grandes rompan el cálculo.
💡 Analogía: Es como una calculadora de bolsillo: si el resultado tiene 200 dígitos, solo muestra "ERROR". BCEWithLogits es como una calculadora científica que sabe manejar números gigantes sin explotar.
💻 Comparación en PyTorch:
# ❌ No recomendado (inestable)
pred = torch.sigmoid(logits)
loss = F.binary_cross_entropy(pred, target)

# ✓ Recomendado (estable)
loss = F.binary_cross_entropy_with_logits(logits, target)
loss = nn.BCEWithLogitsLoss() → Recibe logits (sin sigmoid)

Backpropagation ("propagación hacia atrás") es el algoritmo que calcula cómo debe cambiar cada peso para reducir el error. Usa la regla de la cadena del cálculo para propagar el gradiente desde la salida hasta la entrada.

🧭 ¿Qué es un Gradiente? (Intuición Visual)

El gradiente de una función te dice en qué dirección y qué tan rápido la función crece más rápidamente.

📐 Matemáticamente:
∇L = [∂L/∂w₁, ∂L/∂w₂, ...]

Vector de derivadas parciales respecto a cada peso

∂L/∂w > 0
↑w → ↑Loss
Debemos bajar w
∂L/∂w < 0
↑w → ↓Loss
Debemos subir w

El gradiente apunta "cuesta arriba" →
Nosotros vamos en dirección opuesta

🏔️ Analogía: Descenso de Gradiente = Bajar una Montaña

1. Tocas el suelo alrededor
→ Calculas el gradiente
2. Sientes la pendiente
→ ¿Qué dirección baja más?
3. Das un paso en esa dirección
→ Actualizas los pesos
4. Repites hasta llegar abajo
→ Convergencia (loss mínimo)
Presiona "Iniciar Descenso" para ver la animación

⚡ Forward Pass vs Backward Pass

➡️ FORWARD PASS
  • Propaga los datos hacia adelante
  • Calcula la predicción (ŷ)
  • Computa el loss
  • Guarda valores intermedios para backprop
output = model(x)
loss = criterion(output, y)
⬅️ BACKWARD PASS
  • Propaga el error hacia atrás
  • Calcula ∂L/∂w para cada peso
  • Usa la regla de la cadena
  • Acumula gradientes en .grad
loss.backward()
# model.layer.weight.grad tiene ∂L/∂w

⛓️ La Regla de la Cadena: El Corazón de Backprop

🤔
El gran problema: ¿Cómo sabe la red cuánto cambiar cada peso?
Tenemos miles de pesos. El error está al final de la red.
¿Cómo "culpamos" a cada peso por su parte del error?
📞 Analogía: El Juego del Teléfono
Imagina el juego del teléfono: un mensaje pasa de persona a persona y al final llega distorsionado. ¿Cuánto "culpa" tiene cada persona del error final?
➡️ Forward Pass
El mensaje (datos) viaja de izquierda a derecha, transformándose en cada paso.
⬅️ Backward Pass
La "culpa" (gradiente) viaja de derecha a izquierda, multiplicándose en cada paso.
🔗 La Regla de la Cadena en Una Frase
∂L/∂w = ∂L/∂a × ∂a/∂z × ∂z/∂w
"El gradiente del todo = producto de gradientes locales"
∂L/∂a
"¿Cuánto afecta a al error?"
∂a/∂z
"¿Cuánto afecta z a a?"
∂z/∂w
"¿Cuánto afecta w a z?"
💡 Intuición clave:
Es como preguntar: "Si cambio el peso w un poquito, ¿cuánto cambia el error L?"
La respuesta se obtiene multiplicando los efectos intermedios.
📚 ¿Nuevo en derivadas? Haz clic aquí para un repaso rápido

Una derivada mide cuánto cambia una cosa cuando cambias otra.

Ejemplo cotidiano:
Si conduces a 60 km/h, tu posición cambia 60 km por cada hora.
→ Velocidad = derivada de posición
En redes neuronales:
Si el peso sube 0.1, ¿cuánto sube el loss?
→ ∂L/∂w = derivada del loss
📝 Notación: ¿Qué significa ∂L/∂w?
∂L
∂w
Se lee: "derivada parcial de L respecto a w"
= "Si cambio w un poquito, ¿cuánto cambia L?"
📋 Cheat Sheet: Derivadas Comunes (clic para expandir)
Multiplicación:
f = w·x → ∂f/∂w = x
Suma:
f = a+b → ∂f/∂a = 1
Cuadrado:
f = x² → ∂f/∂x = 2x
ReLU:
ReLU'(x) = 1 si x>0, sino 0
Sigmoid:
σ'(x) = σ(x)(1-σ(x))
MSE Loss:
∂L/∂ŷ = 2(ŷ-y)
💡 Buena noticia: PyTorch calcula todas estas derivadas automáticamente con loss.backward()

🎯 ¿Por Qué Actualizamos los Pesos Así?

🤔 La Gran Pregunta:
Ya sabemos cómo calcular ∂L/∂w con la regla de la cadena.
Pero ¿por qué la fórmula de actualización es w_new = w_old - η × ∂L/∂w?
📌 Paso 1: Nuestro Objetivo
Queremos
Minimizar L
(reducir el error)
→
El Loss L mide qué tan equivocada está la red.
Si L es grande = predicciones malas
Si L es pequeño = predicciones buenas
📌 Paso 2: ¿Qué Nos Dice el Gradiente?
Si ∂L/∂w > 0 (positivo)
Cuando w sube → L sube
El gradiente "apunta" hacia arriba en L
Si ∂L/∂w < 0 (negativo)
Cuando w sube → L baja
El gradiente "apunta" hacia abajo en L
🔑 Insight Clave: El gradiente ∂L/∂w apunta en la dirección donde L crece más rápido
📌 Paso 3: La Lógica del Signo Negativo
El gradiente
∂L/∂w
apunta hacia SUBIR L
pero
Nosotros queremos
BAJAR L
(minimizar el error)
🎯 Solución: ¡Ir en la dirección OPUESTA al gradiente!
wnew = wold − (algo) × ∂L/∂w
El signo menos nos hace ir en contra del gradiente → hacia donde L decrece
📌 Paso 4: ¿Qué Tamaño de Paso? (Learning Rate η)
El gradiente nos dice la dirección, pero ¿cuánto debemos movernos?
η muy grande
🦘
Saltos enormes.
Puede "pasarse" del mínimo y diverger.
η balanceado ✓
🐕
Pasos razonables.
Converge de forma estable.
η muy pequeño
🐌
Pasos diminutos.
Funciona, pero muy lento.
η (eta) = "learning rate" = controla el tamaño del paso.
Valores típicos: 0.001, 0.01, 0.1

🧮 Ejemplo Completo: Mini-red de 2 Capas con ReLU

Presiona "Siguiente Paso" para ver el Forward y Backward pass
💡
¿Por qué Capa 2 no tiene ReLU?
• Capas ocultas (como Capa 1): Usan activación (ReLU, tanh, etc.) para introducir no-linealidad
• Capa de salida para regresión: Sin activación → la predicción puede ser cualquier valor real
• Capa de salida para clasificación: Sigmoid (binaria) o Softmax (multiclase)
➡️ FORWARD PASS
📥 SETUP
x=1.0, w₁=0.5, b₁=0, w₂=2.0, b₂=0, y=0.2
🔮 CAPA 1 (oculta)
z₁ = w₁·x + b₁ = 0.5×1 + 0 = 0.5
↳ ∂z₁/∂w₁ = x = 1.0  (se usará en backward)
a₁ = ReLU(z₁) = ReLU(0.5) = 0.5
↳ ReLU'(z) = 1 si z>0,   0 si z≤0  → ReLU'(0.5) = 1
🎯 CAPA 2 (salida)
z₂ = w₂·a₁ + b₂ = 2×0.5 + 0 = 1.0
↳ ∂z₂/∂w₂ = a₁ = 0.5,   ∂z₂/∂a₁ = w₂ = 2.0
ŷ = z₂ = 1.0 ← sin activación (regresión)
↳ ∂ŷ/∂z₂ = 1  (identidad)
📉 LOSS
L = (ŷ - y)² = (1.0 - 0.2)² = 0.64
↳ ∂L/∂ŷ = 2(ŷ - y) = 2(0.8) = 1.6
⬅️ BACKWARD PASS
1️⃣ ∂L/∂ŷ (derivada del Loss)
L = (ŷ - y)² → derivada = 2(ŷ - y)
= 2(1.0 - 0.2) = 2×0.8 = 1.6
2️⃣ ∂L/∂w₂ (regla de la cadena)
ŷ = w₂·a₁ + b₂
∂(w₂·a₁)/∂w₂ = a₁,   ∂(b₂)/∂w₂ = 0
→ ∂ŷ/∂w₂ = a₁ + 0 = 0.5
∂L/∂w₂ = ∂L/∂ŷ × ∂ŷ/∂w₂ = 1.6 × 0.5 = 0.8
3️⃣ ∂L/∂a₁ (propagando hacia atrás)
ŷ = w₂·a₁ + b₂
∂(w₂·a₁)/∂a₁ = w₂,   ∂(b₂)/∂a₁ = 0
→ ∂ŷ/∂a₁ = w₂ + 0 = 2.0
∂L/∂a₁ = ∂L/∂ŷ × ∂ŷ/∂a₁ = 1.6 × 2.0 = 3.2
4️⃣ ∂L/∂z₁ (a través de ReLU)
a₁ = ReLU(z₁) → ReLU'(z₁) = 1 (porque z₁=0.5>0)
∂L/∂z₁ = ∂L/∂a₁ × ReLU'(z₁) = 3.2 × 1 = 3.2
5️⃣ ∂L/∂w₁ (regla de la cadena final)
z₁ = w₁·x + b₁ → ∂z₁/∂w₁ = x = 1.0
∂L/∂w₁ = ∂L/∂z₁ × ∂z₁/∂w₁ = 3.2 × 1.0 = 3.2
🔄 ACTUALIZACIÓN DE PESOS
¿Y ahora qué hacemos con los gradientes?
El gradiente ∂L/∂wi nos dice "si aumento wi, ¿cuánto sube el error?"
→ Si el gradiente es positivo: aumentar wi aumenta el error → debemos reducir wi
→ Si el gradiente es negativo: aumentar wi reduce el error → debemos aumentar wi
Por eso restamos el gradiente: vamos en dirección contraria al error.
Fórmula del Gradiente Descendente:
winuevo = wiviejo − η × ∂L/∂wi
η (eta) = learning rate = 0.1 → controla qué tan grandes son los pasos (muy grande = inestable, muy pequeño = lento)
w₂ (usamos ∂L/∂w₂ = 0.8)
2.0 − 0.1 × 0.8 = 2.0 − 0.08 = 1.92
w₁ (usamos ∂L/∂w₁ = 3.2)
0.5 − 0.1 × 3.2 = 0.5 − 0.32 = 0.18
b₂ (usamos ∂L/∂b₂ = 1.6)
0 − 0.1 × 1.6 = 0 − 0.16 = −0.16
b₁ (usamos ∂L/∂b₁ = 3.2)
0 − 0.1 × 3.2 = 0 − 0.32 = −0.32
💡 Observa: w₁ cambia más que w₂ (gradiente 3.2 vs 0.8). El error se "amplifica" al propagarse hacia atrás por la regla de la cadena.
⚠️ ¿Y si usáramos Sigmoid en vez de ReLU?
En el paso 4️⃣ multiplicamos por ReLU'(0.5) = 1. ¿Qué pasaría con otras activaciones?
✅ ReLU
Derivada = 1 (si z > 0)
El gradiente pasa intacto
3.2 × 1 = 3.2 ✓
⚠️ Sigmoid
Derivada máxima = 0.25
El gradiente se reduce a ¼
3.2 × 0.25 = 0.8 ↓
⚠️ Tanh
Derivada máxima = 1
Pero cerca de 0, típicamente ~0.4
3.2 × 0.4 = 1.28 ↓
🔴 El Problema del Desvanecimiento del Gradiente
Imagina una red con 10 capas usando Sigmoid. En cada capa multiplicamos por ~0.25:
Gradiente final = gradiente inicial × 0.2510 = gradiente × 0.00000095
¡El gradiente prácticamente desaparece! Las primeras capas casi no aprenden.
✅ Por eso usamos ReLU (y variantes)
ReLU: Derivada = 1 para z > 0. Sin desvanecimiento.
Leaky ReLU: Derivada = 0.01 para z ≤ 0. Evita "neuronas muertas".
GELU/SiLU: Suaves, usados en transformers modernos.
💀 Neuronas Muertas (Dying ReLU)
Si z ≤ 0, entonces ReLU'(z) = 0. El gradiente se bloquea completamente. Si una neurona "muere" (siempre produce valores negativos), nunca se recupera. → Por eso existen Leaky ReLU, PReLU, ELU...

🔥 PyTorch Autograd: ¡Todo Automático!

En la práctica, nunca calculas gradientes a mano. PyTorch construye un grafo computacional durante el forward pass y lo usa para calcular todos los gradientes automáticamente.

📊 ¿Qué es el Grafo Computacional?

Imagina que cada operación matemática que haces es un nodo en un diagrama. PyTorch va "grabando" todas las operaciones mientras calculas el forward pass, creando un grafo dirigido que conecta:

📥 Tensores de Entrada
x, pesos, biases
⚙️ Operaciones
multiplicación, suma, ReLU, etc.
📤 Salida Final
loss (el valor que queremos minimizar)
🎮 Visualización del Grafo (haz clic en "Construir Grafo" para animarlo)
El grafo muestra cómo PyTorch conecta operaciones para calcular gradientes
💡 ¿Por qué es tan útil?
  • No calculas derivadas: PyTorch conoce la derivada de cada operación básica (suma, producto, ReLU, etc.)
  • Regla de la cadena automática: El grafo permite aplicar la regla de la cadena "caminando hacia atrás"
  • Redes arbitrarias: Funciona para cualquier arquitectura, sin importar cuán compleja sea
  • Eficiente: Solo calcula los gradientes que necesitas (los que tienen requires_grad=True)
💻 Código PyTorch Completo
# 1. Definir tensores CON tracking
x = torch.tensor([1.0], requires_grad=True)
w = torch.tensor([0.5], requires_grad=True)

# 2. Forward pass (¡PyTorch graba!)
y_pred = x * w          # Operación guardada
loss = (y_pred - 0.2)**2 # Operación guardada

# 3. Backward (UNA línea!)
loss.backward()  # Calcula TODOS los gradientes

# 4. Acceder a gradientes
print(w.grad)  # → tensor([0.6000])

Backpropagation ("propagación hacia atrás") es el algoritmo que calcula cómo debe cambiar cada peso para reducir el error. Usa la regla de la cadena del cálculo para propagar el gradiente desde la salida hasta la entrada.

📋 Código para copiar y pegar: autograd simple

Este mini-ejemplo muestra cómo PyTorch calcula automáticamente los gradientes de una función sencilla y actualiza un peso siguiendo la dirección de descenso de gradiente.

import torch

# Datos sencillos: x y y_true fijos
x = torch.tensor([2.0])
y_true = torch.tensor([4.0])

# Peso con gradiente
w = torch.tensor([1.0], requires_grad=True)

for step in range(5):
    # Forward: y = w * x
    y_pred = w * x
    loss = (y_pred - y_true).pow(2).mean()  # MSE

    # Backward
    loss.backward()

    print(f"Paso {step} | w = {w.item():.3f} | grad = {w.grad.item():.3f} | loss = {loss.item():.3f}")

    # Actualizar peso manualmente (SGD simple)
    with torch.no_grad():
        w -= 0.1 * w.grad

    # Importante: limpiar gradiente para el siguiente paso
    w.grad.zero_()

🎲 Celda 8: Dropout — Apagando neuronas aleatoriamente (train vs eval)

La regularización son técnicas para evitar el overfitting (sobreajuste): cuando la red memoriza los datos de entrenamiento en lugar de aprender patrones generalizables. Señal de overfitting: loss de entrenamiento ↓ pero loss de validación ↑.

📚
Analogía: Estudiar para un examen

Mal estudiante (overfitting): Memoriza las respuestas exactas del libro de ejercicios. Saca 100% en práctica, pero 40% en el examen real (datos nuevos).

Buen estudiante (generalización): Entiende los conceptos. Saca 85% en práctica y 80% en el examen. ¡Eso es lo que queremos!

🛠️ Técnicas de regularización:
  • Dropout: Fuerza redundancia (la red no puede confiar en neuronas específicas)
  • Early Stopping: Para antes de memorizar
  • Weight Decay (L2): Penaliza pesos grandes
  • Data Augmentation: Más variedad de datos
Weight Decay (L2)
Añade al loss: λ Σ w² → empuja pesos a ser pequeños.
Ejemplo: loss_total = loss_datos + 0.01 Σ w².
AdamW(..., weight_decay=1e-2)
Dropout sin miedo
Durante train apaga p% neuronas (p=0.5 típico). En eval se reactivan todas; PyTorch lo maneja con model.train() / model.eval().
Early Stopping práctico
Guarda el mejor modelo en validación; si en 10 épocas no mejora, detén y restaura el checkpoint.

🎲 Dropout

"Apaga" neuronas aleatorias durante entrenamiento (ej: 50%).

nn.Dropout(p=0.5)

🛑 Early Stopping

Parar cuando el loss de validación empiece a subir.

Guardar mejor modelo, parar si no mejora en N épocas
⚠️
¡Cuidado con el overfitting! Si el loss de entrenamiento baja pero el de validación sube, la red está memorizando.
