📦 ¿Qué es un Batch y por qué importa?

Hasta ahora asumimos que calculamos el loss sobre todo el dataset y luego actualizamos los pesos. Pero con millones de datos, esto es muy lento. La solución: procesar en lotes (batches).

1️⃣
SGD Estocástico
Batch Size = 1
Un punto a la vez
⚡ Rápido pero ruidoso
📦
Mini-Batch
Batch Size = 4-64
Grupo de puntos
✨ Balance ideal
📊
Full Batch
Batch Size = Todo
Dataset completo
🎯 Estable pero lento
¿Cómo afecta al Loss?
  • SGD (1): Loss calculado con 1 punto → muy variable, curva zigzagueante
  • Mini-batch (8): Loss promediado sobre 8 puntos → más suave, buen balance
  • Full batch: Loss exacto del dataset completo → curva muy suave, pero lento

💡 En la práctica se usa mini-batch (32-256). Prueba diferentes tamaños en el simulador y observa la curva de loss.

💡 Dos Formas de Resolver Problemas No-Lineales

A veces el problema no es linealmente separable con las entradas originales (x₁, x₂). Hay dos formas de resolverlo:

🔧 Feature Engineering
Tú diseñas las transformaciones.
Creas nuevas entradas como x², sin(x), x₁×x₂, etc.
Pros: Redes pequeñas, entrena rápido
Cons: Requiere conocimiento del dominio
🧠 Representation Learning
La red descubre las transformaciones.
Las capas ocultas aprenden features útiles automáticamente.
Pros: Descubre patrones que no imaginaste
Cons: Necesita más datos y computación
Mismo problema, dos enfoques:
Feature Engineering
x₁, x₂ → [x₁², x₂²] → Red simple
vs
Representation Learning
x₁, x₂ → Red profunda → aprende x₁², x₂²
🔵 Patrón: Círculos

Frontera de decisión: x₁² + x₂² = r²

🔧 Feature Engineering:
Agregas x₁², x₂² → 1 capa, 2 neuronas
🧠 Representation Learning:
Solo x₁, x₂ → 2 capas, 8 neuronas, 500 épocas
🌀 Patrón: Espiral

Depende de ángulo + distancia → muy no-lineal

🔧 Feature Engineering:
Agregas x₁x₂, sin(x₁), sin(x₂) → 2 capas, 4 neuronas
🧠 Representation Learning:
Solo x₁, x₂ → 3 capas, 16 neuronas, 1000+ épocas
🔑 El Secreto del Representation Learning: Las Funciones de Activación
¿Cómo puede una red "inventar" transformaciones como x² o sin(x)? Gracias a las activaciones no lineales (ReLU, tanh, etc.):
Sin activación (solo lineal):
100 capas = 1 capa lineal
Solo puede aprender líneas rectas
Con ReLU entre capas:
Cada capa "dobla" el espacio
Puede aproximar cualquier función
💡 Teorema de Aproximación Universal: Una red con 1 capa oculta y activación no lineal puede aproximar cualquier función continua (con suficientes neuronas). ¡Las activaciones son la clave!
🎓 En la práctica: Se combinan ambos. Usas features que conoces del dominio + capas profundas que descubren patrones que no imaginaste. Deep Learning = Representation Learning extremo (redes con muchas capas que aprenden features cada vez más abstractas).

🎓 ¿Qué significan estas métricas?
📦 División del Dataset
Train (80%): Datos que la red usa para aprender.
Validation (20%): Datos que la red nunca ve durante el entrenamiento, solo para evaluar.
🤔 ¿Por qué separar?
Si solo mides en datos de entrenamiento, no sabes si la red generaliza o solo memoriza. El set de validación simula "datos nuevos".
ÉPOCA
Una pasada completa por todo el dataset de train
TRAIN LOSS
Error promedio en datos de entrenamiento
VAL LOSS
Error en datos "nuevos" (validación)
TRAIN ACC
% de aciertos en entrenamiento
VAL ACC
% de aciertos en validación (el que importa!)
⚠️ Señales de Overfitting (Sobreajuste)
Train Loss ↓ pero Val Loss ↑ = La red memoriza en vez de aprender patrones.
Train Acc alto, Val Acc bajo = Funciona perfecto en datos conocidos, falla en nuevos.
💡 Solución: Menos neuronas, menos capas, o técnicas de regularización (Dropout, L2).
