¿Qué es una LSTM?

Introducción a las redes de memoria a largo-corto plazo

Una LSTM (Long Short-Term Memory) es un tipo especial de Red Neuronal Recurrente (RNN) diseñada específicamente para aprender dependencias a largo plazo en datos secuenciales. Fue propuesta por Sepp Hochreiter y Jürgen Schmidhuber en 1997 para resolver el problema fundamental del gradiente desvaneciente que afectaba a las RNN convencionales.

💡 Analogía Intuitiva

Imagina que estás leyendo un libro. Una RNN simple es como un lector con memoria de pez: para cuando llega al capítulo 10, ya olvidó los personajes del capítulo 1. Una LSTM es como un lector con una libreta de notas: puede decidir qué información guardar, qué actualizar y qué olvidar, manteniendo la información relevante durante toda la lectura.

¿Qué tipo de datos procesan las LSTM?

Las LSTM son ideales para cualquier dato donde el orden importa:

  • Texto y lenguaje natural: Traducción automática, generación de texto, análisis de sentimiento
  • Series temporales: Predicción de precios, pronóstico meteorológico, datos de sensores IoT
  • Audio y voz: Reconocimiento de voz, generación de música
  • Video: Clasificación de acciones, generación de descripciones
  • Datos biomédicos: Señales ECG, EEG, secuencias genómicas

Evolución Histórica

1986

RNN propuesta por Rumelhart — Primeras redes recurrentes con backpropagation through time.

1991

Problema del gradiente identificado por Hochreiter — Se demuestra formalmente que las RNN no pueden aprender dependencias a largo plazo.

1997

LSTM inventada — Hochreiter & Schmidhuber publican "Long Short-Term Memory", introduciendo el mecanismo de compuertas.

2000

Forget Gate añadida por Gers et al. — Se agrega la compuerta de olvido, completando la arquitectura moderna.

2014

GRU propuesta por Cho et al. — Variante simplificada con solo 2 compuertas.

2017

Transformer (Attention Is All You Need) — Nuevo paradigma basado en atención, pero las LSTM siguen vigentes para muchas aplicaciones.

El Problema: Gradiente Desvaneciente

Por qué las RNN simples fallan con secuencias largas

¿Cómo funciona una RNN Simple?

Una RNN básica procesa secuencias paso a paso, manteniendo un estado oculto $h_t$ que se actualiza en cada paso temporal:

Ecuación de una RNN Simple
$$h_t = \tanh(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b)$$

Donde $W_{hh}$ son los pesos recurrentes, $W_{xh}$ los pesos de entrada, y $b$ el sesgo.

El Problema Fundamental

Durante el entrenamiento, al hacer Backpropagation Through Time (BPTT), los gradientes se multiplican repetidamente por la matriz $W_{hh}$ en cada paso temporal. Si la secuencia tiene longitud $T$, el gradiente es proporcional a:

Flujo del Gradiente
$$\frac{\partial L}{\partial h_0} \propto \prod_{t=1}^{T} W_{hh} \cdot \text{diag}(\tanh'(h_t))$$

⚠️ Gradiente Desvaneciente (Vanishing Gradient)

Si los valores propios de $W_{hh}$ son menores a 1, al multiplicar muchas veces el gradiente se vuelve exponencialmente pequeño → los pesos no se actualizan → la red no aprende relaciones distantes.

💥 Gradiente Explosivo (Exploding Gradient)

Si los valores propios son mayores a 1, el gradiente crece exponencialmente → inestabilidad numérica → NaN en el entrenamiento. Este problema se puede mitigar con gradient clipping, pero el gradiente desvaneciente requiere un cambio de arquitectura: las LSTM.

Considera esta oración:

Para predecir "francés", la red necesita recordar "Francia" que aparece ~25 palabras antes. Una RNN simple no puede mantener esta conexión; una LSTM sí.

Arquitectura de la Celda LSTM

Los componentes internos que permiten la memoria a largo plazo

La innovación clave de las LSTM es la introducción de un estado de celda $C_t$ — una "autopista de información" que recorre toda la secuencia con modificaciones mínimas, permitiendo que los gradientes fluyan sin degradarse.

Componentes Principales

🛣️
Estado de Celda ($C_t$)

La "autopista de información". Fluye a través de toda la cadena con solo operaciones lineales (suma y multiplicación elemento a elemento), permitiendo que el gradiente se propague sin degradarse.

🚫
Compuerta de Olvido ($f_t$)

Decide qué información del estado de celda anterior debe descartarse. Produce valores entre 0 (olvidar completamente) y 1 (recordar todo).

📥
Compuerta de Entrada ($i_t$)

Decide qué nueva información se almacenará en el estado de celda. Trabaja con un candidato $\tilde{C}_t$ generado por una capa tanh.

📤
Compuerta de Salida ($o_t$)

Decide qué parte del estado de celda se emitirá como el estado oculto $h_t$, que es la salida de la celda.

Diagrama de la Celda LSTM

Diagrama completo de una celda LSTM mostrando el estado de celda (amarillo), compuerta de olvido (rojo), compuerta de entrada (azul) y compuerta de salida (verde).

🔑 Clave de la Arquitectura

El estado de celda $C_t$ es la "autopista de información" — fluye de izquierda a derecha con solo operaciones lineales (multiplicación y suma punto a punto). Esto permite que los gradientes fluyan sin degradarse, resolviendo el problema del gradiente desvaneciente.

Las Tres Compuertas en Detalle

Mecanismo interno de cada compuerta con analogías y ecuaciones

🚫 Paso 1: Compuerta de Olvido (Forget Gate)

La primera decisión que toma la celda LSTM es qué información descartar del estado de celda anterior. Esta decisión es tomada por una capa sigmoide llamada "compuerta de olvido".

Forget Gate
$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$

📖 Analogía: El Editor de un Periódico

Imagina un editor que revisa las noticias del día anterior. La compuerta de olvido es como el editor decidiendo: "¿Esta noticia sigue siendo relevante hoy?" Si $f_t = 1$, la noticia se conserva; si $f_t = 0$, se descarta para hacer espacio a noticias nuevas.

Ejemplo concreto en NLP: Al procesar texto y encontrar un nuevo sujeto en la oración, la compuerta de olvido puede "olvidar" el género del sujeto anterior para reemplazarlo con el nuevo.

📥 Paso 2: Compuerta de Entrada (Input Gate)

El siguiente paso tiene dos partes: (1) una capa sigmoide decide qué valores actualizar, y (2) una capa tanh crea un vector de nuevos valores candidatos.

Input Gate + Candidato
$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$$ $$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$$

📖 Analogía: El Periodista

Si la compuerta de olvido es el editor eliminando noticias viejas, la compuerta de entrada es el periodista escribiendo noticias nuevas. El sigmoide ($i_t$) decide la importancia de cada noticia, y el tanh ($\tilde{C}_t$) genera el contenido de la noticia.

🔄 Paso 3: Actualización del Estado de Celda

Ahora combinamos los dos pasos anteriores para crear el nuevo estado de celda:

Actualización del Estado de Celda
$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$

Donde $\odot$ es la multiplicación elemento a elemento (producto de Hadamard). Primero multiplicamos el estado anterior por $f_t$ para olvidar lo que decidimos olvidar, y luego sumamos $i_t \odot \tilde{C}_t$ para agregar la nueva información.

📤 Paso 4: Compuerta de Salida (Output Gate)

Finalmente, la celda decide qué emitir como salida. La salida se basa en el estado de celda, pero filtrado:

Output Gate + Estado Oculto
$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$$ $$h_t = o_t \odot \tanh(C_t)$$

📖 Analogía: El Portavoz

La compuerta de salida es como el portavoz de una empresa: la empresa (estado de celda) tiene mucha información interna, pero el portavoz decide cuánto y qué comunicar al público (estado oculto). No toda la información almacenada es relevante para la salida actual.

🧮 Resumen de Parámetros

Para una LSTM con tamaño de entrada $n$ y tamaño oculto $m$, el número total de parámetros es:

$4 \times [(n + m) \times m + m] = 4m(n + m + 1)$

El factor 4 corresponde a las 4 transformaciones lineales: $W_f$, $W_i$, $W_C$ y $W_o$.

# Entrada actual
x_t = [0.5, 0.3]

concat = [0.1, -0.2, 0.5, 0.3]

Paso 1: Forget Gate

Cálculo de $f_t$
$$f_t = \sigma(W_f \cdot [0.1, -0.2, 0.5, 0.3] + b_f)$$ $$f_t = \sigma([0.32, -0.15]) = [0.58, 0.46]$$

Interpretación: La primera dimensión se retiene al 58%, la segunda al 46%.

Paso 2: Input Gate + Candidato

Cálculo de $i_t$ y $\tilde{C}_t$
$$i_t = \sigma(W_i \cdot \text{concat} + b_i) = [0.72, 0.35]$$ $$\tilde{C}_t = \tanh(W_C \cdot \text{concat} + b_C) = [0.41, -0.68]$$

Paso 3: Actualizar Estado de Celda

Cálculo de $C_t$
$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$ $$C_t = [0.58, 0.46] \odot [0.8, -0.5] + [0.72, 0.35] \odot [0.41, -0.68]$$ $$C_t = [0.464, -0.230] + [0.295, -0.238]$$ $$C_t = [0.759, -0.468]$$

Paso 4: Output Gate + Estado Oculto

Cálculo de $o_t$ y $h_t$
$$o_t = \sigma(W_o \cdot \text{concat} + b_o) = [0.63, 0.81]$$ $$h_t = o_t \odot \tanh(C_t) = [0.63, 0.81] \odot \tanh([0.759, -0.468])$$ $$h_t = [0.63, 0.81] \odot [0.639, -0.436] = [0.403, -0.353]$$

✅ Resultado Final

Nuevo estado de celda: $C_t = [0.759, -0.468]$ — contiene la memoria a largo plazo actualizada.

Nuevo estado oculto: $h_t = [0.403, -0.353]$ — es la salida de la celda que se pasa al siguiente paso temporal y/o a la capa de salida.

Variantes de LSTM

GRU, Peephole Connections y Bidireccional

GRU (Gated Recurrent Unit)

Propuesta por Cho et al. (2014), la GRU es una variante simplificada que combina las compuertas de olvido y entrada en una sola "compuerta de actualización" y fusiona el estado de celda con el estado oculto.

Ecuaciones de la GRU
$$z_t = \sigma(W_z \cdot [h_{t-1}, x_t]) \quad \text{(Update Gate)}$$ $$r_t = \sigma(W_r \cdot [h_{t-1}, x_t]) \quad \text{(Reset Gate)}$$ $$\tilde{h}_t = \tanh(W \cdot [r_t \odot h_{t-1}, x_t])$$ $$h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$$
Característica LSTM GRU
Compuertas 3 (forget, input, output) 2 (update, reset)
Estados 2 ($C_t$ y $h_t$) 1 ($h_t$ solamente)
Parámetros $4m(n+m+1)$ $3m(n+m+1)$
Rendimiento Mejor en secuencias muy largas Similar en secuencias cortas/medianas
Velocidad Más lenta (más parámetros) Más rápida (~25% menos cómputo)
Cuándo usar Dependencias muy largas, más datos Datasets pequeños, tiempo limitado

Peephole Connections

Propuestas por Gers & Schmidhuber (2000), las conexiones peephole permiten que las compuertas "miren" directamente al estado de celda $C_{t-1}$:

LSTM con Peephole
$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + \textcolor{orange}{W_{pf} \cdot C_{t-1}} + b_f)$$ $$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + \textcolor{orange}{W_{pi} \cdot C_{t-1}} + b_i)$$ $$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + \textcolor{orange}{W_{po} \cdot C_t} + b_o)$$

💡 ¿Cuándo usar Peephole?

Las peephole connections son útiles cuando se necesita un control preciso del timing, como en tareas de generación de música o procesamiento de señales con patrones rítmicos.

LSTM Bidireccional (BiLSTM)

Una BiLSTM procesa la secuencia en ambas direcciones (izquierda→derecha y derecha→izquierda), capturando tanto el contexto pasado como el futuro:

BiLSTM: las salidas forward y backward se concatenan, produciendo un vector de dimensión $2m$ por cada paso temporal.

🌊 Usos típicos de BiLSTM

NER (Named Entity Recognition): Para etiquetar "Madrid" como ciudad, necesitas saber que antes dice "vive en" y después dice "desde 2020".

Análisis de sentimiento: La frase "No creo que sea malo" requiere contexto completo.

Traducción automática (encoder): El encoder de un sistema seq2seq a menudo es bidireccional.

Implementación en PyTorch

Código completo y funcional con explicaciones línea por línea

LSTM Básica para Clasificación de Secuencias

Python 
import torch
import torch.nn as nn

class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, output_dim,
                 n_layers=2, bidirectional=True, dropout=0.3):
        super().__init__()

        # Capa de embedding: convierte índices de palabras en vectores densos
        self.embedding = nn.Embedding(vocab_size, embed_dim)

        # LSTM: el corazón del modelo
        self.lstm = nn.LSTM(
            input_size=embed_dim,       # Dimensión de cada embedding
            hidden_size=hidden_dim,     # Dimensión del estado oculto
            num_layers=n_layers,        # Capas LSTM apiladas
            bidirectional=bidirectional,# Procesar en ambas direcciones
            dropout=dropout,            # Dropout entre capas LSTM
            batch_first=True           # Input shape: (batch, seq, feature)
        )

        # Factor 2 si es bidireccional
        direction_factor = 2 if bidirectional else 1

        # Capa fully-connected para clasificación
        self.fc = nn.Linear(hidden_dim * direction_factor, output_dim)
        self.dropout = nn.Dropout(dropout)

    def forward(self, text):
        # text shape: (batch_size, seq_length)

        embedded = self.dropout(self.embedding(text))
        # embedded shape: (batch_size, seq_length, embed_dim)

        output, (hidden, cell) = self.lstm(embedded)
        # output shape: (batch, seq_len, hidden_dim * num_directions)
        # hidden shape: (n_layers * num_directions, batch, hidden_dim)
        # cell shape:   (n_layers * num_directions, batch, hidden_dim)

        # Para clasificación, usamos el último hidden state
        # Concatenamos forward y backward del último layer
        if self.lstm.bidirectional:
            hidden_cat = torch.cat((hidden[-2], hidden[-1]), dim=1)
        else:
            hidden_cat = hidden[-1]

        # Clasificación final
        output = self.dropout(hidden_cat)
        return self.fc(output)

Entrenamiento

Python 
# Configuración
VOCAB_SIZE = 25000
EMBED_DIM = 128
HIDDEN_DIM = 256
OUTPUT_DIM = 2  # Binario: positivo/negativo

model = LSTMClassifier(VOCAB_SIZE, EMBED_DIM, HIDDEN_DIM, OUTPUT_DIM)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss()

# Loop de entrenamiento
def train_epoch(model, dataloader, optimizer, criterion):
    model.train()
    total_loss = 0

    for batch_text, batch_labels in dataloader:
        optimizer.zero_grad()
        predictions = model(batch_text)
        loss = criterion(predictions, batch_labels)
        loss.backward()

        # Gradient clipping para evitar gradiente explosivo
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

        optimizer.step()
        total_loss += loss.item()

    return total_loss / len(dataloader)

⚡ Tip Importante: Gradient Clipping

clip_grad_norm_ es crucial para entrenar LSTMs. Aunque las LSTM resuelven el gradiente desvaneciente, el gradiente explosivo aún puede ocurrir. Un max_norm de 1.0 a 5.0 funciona bien en la mayoría de casos.

LSTM para Series Temporales (Regresión)

Python 
class LSTMTimeSeries(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_layers, output_dim):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        # x shape: (batch, sequence_length, input_features)

        lstm_out, (h_n, c_n) = self.lstm(x)
        # Usamos la salida del último paso temporal
        last_output = lstm_out[:, -1, :]  # (batch, hidden_dim)
        prediction = self.fc(last_output)
        return prediction

# Ejemplo: predecir el siguiente valor de una serie
model = LSTMTimeSeries(
    input_dim=1,      # Una feature (ej: precio)
    hidden_dim=64,    # 64 neuronas ocultas
    num_layers=2,     # 2 capas LSTM apiladas
    output_dim=1      # Predecir un valor
)

Implementación en TensorFlow/Keras

La misma arquitectura con la API de alto nivel de Keras

Clasificación de Texto con LSTM

Python 
import tensorflow as tf
from tensorflow.keras import layers, models

def build_lstm_classifier(vocab_size, embed_dim, hidden_dim, output_dim):
    model = models.Sequential([
        # Capa de embedding
        layers.Embedding(vocab_size, embed_dim),

        # LSTM Bidireccional
        # return_sequences=True solo si apilamos otra LSTM encima
        layers.Bidirectional(
            layers.LSTM(hidden_dim, return_sequences=True, dropout=0.3)
        ),

        # Segunda capa LSTM (no necesita return_sequences)
        layers.Bidirectional(
            layers.LSTM(hidden_dim, dropout=0.3)
        ),

        # Clasificación
        layers.Dense(hidden_dim, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(output_dim, activation='softmax')
    ])
    return model

model = build_lstm_classifier(
    vocab_size=25000,
    embed_dim=128,
    hidden_dim=256,
    output_dim=2
)

model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

model.summary()

def build_lstm_timeseries(seq_length, n_features):
    model = models.Sequential([
        layers.LSTM(128, return_sequences=True,
                    input_shape=(seq_length, n_features)),
        layers.Dropout(0.2),

        layers.LSTM(64, return_sequences=False),
        layers.Dropout(0.2),

        layers.Dense(32, activation='relu'),
        layers.Dense(1)  # Regresión: sin activación
    ])

    model.compile(optimizer='adam', loss='mse', metrics=['mae'])
    return model

# Uso: 60 pasos temporales, 1 feature
model = build_lstm_timeseries(seq_length=60, n_features=1)

📝 PyTorch vs Keras: ¿Cuál elegir?

PyTorch: Mayor control, ideal para investigación, depuración más fácil con ejecución eager.

Keras: Más rápido para prototipar, API más limpia, mejor para producción con TensorFlow Serving.

Ambos producen resultados equivalentes. La elección es cuestión de preferencia y ecosistema.

Pipeline completo desde datos crudos hasta predicción

Paso 1: Preparación de Datos

Python 
import numpy as np
import pandas as pd
from sklearn.preprocessing import MinMaxScaler

# Cargar datos
df = pd.read_csv('stock_prices.csv')
prices = df['Close'].values.reshape(-1, 1)

# Normalizar entre 0 y 1
scaler = MinMaxScaler()
prices_scaled = scaler.fit_transform(prices)

# Crear secuencias: ventana de 60 días para predecir el día 61
def create_sequences(data, seq_length=60):
    X, y = [], []
    for i in range(seq_length, len(data)):
        X.append(data[i - seq_length:i, 0])
        y.append(data[i, 0])
    return np.array(X), np.array(y)

X, y = create_sequences(prices_scaled, seq_length=60)

# Reshape para LSTM: (samples, timesteps, features)
X = X.reshape(X.shape[0], X.shape[1], 1)

# Split train/test (80/20)
split = int(len(X) * 0.8)
X_train, X_test = X[:split], X[split:]
y_train, y_test = y[:split], y[split:]

Paso 2: Modelo

Python 
import torch
import torch.nn as nn

class StockLSTM(nn.Module):
    def __init__(self):
        super().__init__()
        self.lstm1 = nn.LSTM(input_size=1, hidden_size=128,
                             batch_first=True, dropout=0.2, num_layers=2)
        self.fc = nn.Sequential(
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(64, 1)
        )

    def forward(self, x):
        lstm_out, _ = self.lstm1(x)
        last_hidden = lstm_out[:, -1, :]  # Último paso temporal
        return self.fc(last_hidden)

model = StockLSTM()

LSTM vs Transformers: ¿Las LSTM están muertas?

🤔 ¡No! Las LSTM siguen siendo relevantes

Aunque los Transformers dominan en NLP desde 2017, las LSTM siguen siendo preferidas en:

  • Recursos limitados: Las LSTM requieren mucha menos memoria y cómputo.
  • Datos pequeños: Los Transformers necesitan enormes cantidades de datos para brillar.
  • Tiempo real/streaming: Las LSTM procesan datos secuencialmente de forma natural.
  • Series temporales: Para series con pocas variables, las LSTM son competitivas.
  • Edge computing: Dispositivos IoT, móviles, sistemas embebidos.
