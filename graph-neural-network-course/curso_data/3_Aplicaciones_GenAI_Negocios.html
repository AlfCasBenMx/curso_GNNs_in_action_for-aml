Flujos sugeridos (paso a paso)

Due Diligence / Data Room

  1. Ingesta docs (contratos, estados financieros) → OCR/limpieza → metadata (fecha, confidencialidad)
  2. Indexar con embeddings + search híbrido
  3. Preguntas guiadas y plantillas de riesgos
  4. RAG con citas obligatorias → resumen ejecutivo
  5. Revisión humana y checklist de hallazgos

Modelo financiero asistido

  1. Conectar hoja de cálculo (lectura sola) y diccionario de tabs
  2. Validar supuestos: detectar celdas extremas o inconsistentes
  3. Generar escenarios (base/bajo/alto) con justificación
  4. RAG sobre inputs y notas → memo ejecutivo
  5. HIL: analista confirma cambios antes de escribir de vuelta

Research y comparables

  1. Extraer tickers/peers desde brief del analista
  2. Consultar fuentes aprobadas (filtrado dominios) y almacenar citas
  3. Normalizar múltiplos (EV/EBITDA, P/E) y fechas
  4. Generar tabla de comps + narrativas de variación
  5. Exportar a deck con secciones estándar

AML / Riesgo

  1. Screening PEP/sanciones → clasificación riesgo
  2. RAG sobre alertas previas y políticas internas
  3. Explicación de alerta + evidencia con enlaces
  4. Generar borrador de SAR/ROS con plantilla
  5. HIL obligatorio antes de envío

Diseño de MVP y ROI

Cómo justificar el caso

Hipótesis de valor

  • Ahorro de tiempo por interacción (min/operación)
  • Deflexión de tickets o reducción AHT
  • Mejora en conversión/CTR en campañas
  • Calidad percibida (CSAT/NPS) y consistencia

Medición

  • Grupo control vs. tratamiento
  • Revisión humana de precisión factual
  • Tiempo ahorrado * costo hora = beneficio
  • Uso efectivo: sesiones, completitud, caídas
✅ Pilota con 1-2 flujos, mide 2 semanas, ajusta prompts y retrieval; luego escala.

Gestión de Conocimiento (RAG)

Calidad de datos primero

Buenas prácticas de ingesta

  • Limpieza de duplicados y versiones
  • Chunking con contexto (títulos, jerarquía)
  • Metadatos: fecha, owner, confidencialidad
  • Embeddings adecuados al idioma

Recuperación eficaz

  • Hybrid search: vector + keyword
  • Re-ranking para precisión
  • Top-k moderado (3-5) + contexto de citas
  • Evaluar recall/precision con conjuntos etiquetados
🧪 Evalúa mensualmente la calidad de retrieval (nDCG, recall@k) y refresca embeddings si cambian los datos.
6

Guardrails y Riesgos

Operar con seguridad

Riesgos

Alucinaciones, PII leakage, contenido tóxico, acciones erróneas de agentes, falta de citas.

Controles

Clasificador toxicity/PII, RAG con citas, límites de herramientas, revisión humana en flujos críticos, logs firmados.

Métricas

Tasa de alucinación, % respuestas con cita, % prompts bloqueados, tiempo de revisión humana.

Experiencia de Usuario

Diseño conversacional y feedback

UX conversacional

  • Personas y tono alineados a la marca
  • Mensajes de error útiles, no genéricos
  • Controles rápidos: botones, intents sugeridos
  • Mostrar fuentes y pasos usados

Feedback y mejora

  • Botón de thumbs + razón
  • Canal de reporte de errores sensibles
  • Bucles de re-etiquetado con ejemplos reales
  • AB testing de prompts y plantillas

Change Management

Métricas de adopción

  • Usuarios activos/semana
  • Sesiones completadas / drop-off
  • Feedback positivo vs. negativo
