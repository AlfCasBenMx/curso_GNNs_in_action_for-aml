⚙️ Matrices de pesos (pre-entrenadas)

Fórmula de actualización de GraphSAGE:

$$h_v^{(\ell+1)} = \sigma\!\left(\underbrace{W_1}_{\text{2×3}}\, \underbrace{h_v^{(\ell)}}_{\text{3×1}} + \underbrace{W_2}_{\text{2×3}}\, \underbrace{m_i^{(\ell)}}_{\text{3×1}}\right)$$

¿Por qué las matrices son 2×3? El tamaño viene de la transformación que necesitamos:
• Los vectores de entrada ($h_v$ y $m_i$) tienen 3 dimensiones (las features de cada nodo) → eso fija el número de columnas de $W$ en 3.
• Queremos que el embedding de salida tenga 2 dimensiones (la capa oculta que elegimos) → eso fija el número de filas de $W$ en 2.
• Resultado: $W_{2\times3} \cdot h_{3\times1} = \text{vector}_{2\times1}$.

En PubMed real: $W$ sería 64×500 (500 features → 64 dimensiones ocultas). Aquí usamos 2×3 para poder seguir cada multiplicación a mano.

$W_1$ transforma la info propia del nodo ($h_v$). $W_2$ transforma el resumen del vecindario ($m_i$). El modelo aprende ambas matrices durante el entrenamiento — aquí las mostramos con valores fijos de ejemplo.

🔺 Agregador Max Pooling — Paso a paso

⚠️ Clave: Max pooling en GraphSAGE NO es simplemente tomar el máximo de los embeddings crudos. Primero se pasa cada vecino por una mini red neuronal $\phi$ (aprendida), y luego se toma el máximo componente a componente de las salidas transformadas.

$W_\phi$ — Pesos de la red $\phi$: transformación aprendida antes del max (3×3 + bias)

¿Qué es esta matriz? Es $W_\phi$, los pesos de la red neuronal $\phi$ que se aplica a cada vecino por separado antes de tomar el máximo. La transformación completa es:
$$\phi(h_j) = \text{ReLU}(W_\phi \cdot h_j + b_\phi)$$ • Las filas p₁, p₂, p₃ son las 3 filas de $W_\phi$ (una por cada neurona de salida).
• Las columnas d₁, d₂, d₃ corresponden a las 3 dimensiones del vector de entrada $h_j$.
• Como la entrada tiene 3 dimensiones y la salida también 3, $W_\phi$ es 3×3. El bias $b_\phi$ se suma después del producto matricial.

Es una capa densa estándar (fully-connected + ReLU). El modelo aprende $W_\phi$ y $b_\phi$ durante el entrenamiento para extraer las características más relevantes de cada vecino antes de agregar con max.
