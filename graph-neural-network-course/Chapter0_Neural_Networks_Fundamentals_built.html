Todo empieza con la neurona artificial, inspirada en las neuronas biológicas del cerebro. Es una función matemática que:

  • Recibe múltiples entradas (números)
  • Multiplica cada entrada por un peso (importancia)
  • Suma todo y añade un bias (ajuste fino)
  • Aplica una función de activación
  • Produce una salida (predicción)
🧠
Analogía: La neurona como un juez aprobando préstamos

Imagina un juez que decide si aprobar un préstamo. Recibe datos (ingresos=$5000, historial crediticio=750, deudas=$10000). Cada dato tiene un "peso" en su decisión:

  • Ingresos: peso=+0.8 (muy importante)
  • Historial: peso=+0.5 (importante)
  • Deudas: peso=-0.6 (penaliza)

Calcula: (0.8×5000) + (0.5×750) + (-0.6×10000) + bias = ¿Aprobar o rechazar?

Anatomía de una Neurona

📐 La Fórmula
ŷ = σ( w₁·x₁ + w₂·x₂ + w₃·x₃ + b )
● x = entradas ● w = pesos ● b = bias ● σ = activación

🔍 Desglosando cada componente:

1. Entradas (x): Los datos crudos
Ej: edad=25, salario=50000
2. Pesos (w): Importancia de cada entrada
Valores ajustables (se aprenden)
3. Bias (b): Ajuste fino/offset
Desplaza el umbral de activación
4. Activación (σ): Transforma la suma
Añade no-linealidad

📋 Código para copiar y pegar: una neurona en Python

Asumiendo que ya copiaste y ejecutaste el Paso 0 (imports), este bloque muestra cómo calcular la salida de una neurona con 3 entradas, pesos y bias, y luego pasarla por una función de activación sigmoide. 

import math

      # Entradas (x)
      x1, x2, x3 = 0.8, 0.5, 0.2  # por ejemplo: normalizadas en [0, 1]

      # Pesos (w) y bias (b)
      w1, w2, w3 = 0.9, 0.4, -0.6
      b = 0.1

      def sigmoid(z: float) -> float:
        return 1 / (1 + math.exp(-z))

      # Suma ponderada (logit)
      z = w1 * x1 + w2 * x2 + w3 * x3 + b

      # Salida de la neurona después de la activación
      y_hat = sigmoid(z)

      print(f"z = {z:.3f}")
      print(f"σ(z) = {y_hat:.3f}")

Puedes cambiar los valores de x1, x2, x3 y de los pesos w1, w2, w3 para ver cómo afecta a la salida de la neurona.

💡
¡Punto clave! Los pesos (w) y bias (b) son los únicos parámetros que la red APRENDE. Al inicio se inicializan aleatoriamente (ej: valores pequeños cerca de 0), y durante el entrenamiento se van ajustando mediante backpropagation para minimizar el error.

La regresión lineal es la red neuronal más simple: 1 neurona, sin función de activación. ¿Por qué? Porque la salida es directamente una combinación lineal de las entradas.

La red aprende w (cuánto vale cada m²) y b (precio base). Por ejemplo: w=2500, b=50000 significa "$50k base + $2.5k por cada m²".

Usamos la función Sigmoid (σ) que "aplasta" cualquier número al rango [0, 1]:

✨ La magia del Sigmoid:

  • Entrada muy negativa → salida ≈ 0
  • Entrada muy positiva → salida ≈ 1
  • Entrada = 0 → salida = 0.5

¿Qué está haciendo realmente la neurona?
Está dibujando una línea (o plano) divisoria en el espacio de datos. Los puntos de un lado → Clase 0, los del otro lado → Clase 1. Esto se llama frontera de decisión.

📋 Código para copiar y pegar: clasificación binaria con Sigmoid

Partiendo de los imports del Paso 0, este ejemplo crea un dataset sencillo de casas etiquetadas como cara (1) o barata (0) según un umbral de precio, y entrena una neurona con activación Sigmoid para aproximar esa clasificación. 

# Datos de ejemplo: m² y etiqueta cara (1) / barata (0)
x_m2 = torch.tensor([[40.], [60.], [80.], [100.], [120.], [150.]] )
precios = 2.5 * x_m2 + 50  # modelo "real" oculto

umbral_precio = 260  # >= 260k -> cara (1), si no barata (0)
labels = (precios >= umbral_precio).float()

print("m²  precio(k)  etiqueta")
for x, p, y in zip(x_m2, precios, labels):
    print(f"{x.item():4.0f}   {p.item():7.1f}     {int(y.item())}")

# Modelo logístico: 1 neurona con Sigmoid
modelo_clf = nn.Sequential(
    nn.Linear(1, 1),
    nn.Sigmoid()
)

criterio = nn.BCELoss()  # Binary Cross-Entropy
opt = torch.optim.SGD(modelo_clf.parameters(), lr=1e-2)

for epoca in range(2000):
    y_pred = modelo_clf(x_m2)
    loss = criterio(y_pred, labels)

    opt.zero_grad()
    loss.backward()
    opt.step()

print("\nP(x es 'cara' | m²):")
with torch.no_grad():
    probs = modelo_clf(x_m2)
    for x, p, prob in zip(x_m2, precios, probs):
        decision = 1 if prob.item() >= 0.5 else 0
        print(f"{x.item():4.0f} m²  precio≈{p.item():6.1f}k  σ(z)={prob.item():.2f}  -> pred={decision}")

La Frontera de Decisión

¿Cuándo funciona una línea recta?

✅ Al terminar esta sección deberías poder:

  • Visualizar la frontera de decisión de una neurona en 2D.
  • Relacionar la ecuación z = 0 con una recta (o plano) en el espacio.
  • Entender cómo cambian las fronteras al modificar pesos y bias.

Una neurona con Sigmoid puede clasificar datos que son linealmente separables: cuando puedes dibujar una línea recta que los divide perfectamente.

🧠

¿Cómo pasamos de regresión a clasificación?

  • Primero, la neurona hace una combinación lineal de las entradas: z = w₁·x₁ + w₂·x₂ + b.
  • Luego aplica la Sigmoid y produce un valor entre 0 y 1: σ(z). Si interpretamos σ(z) como "probabilidad de ser clase 1", tenemos un modelo de clasificación.
  • Con un umbral (por ejemplo 0.5) decidimos la clase: si σ(z) ≥ 0.5 → Clase 1, si σ(z) < 0.5 → Clase 0.

El lugar donde la neurona está "justo en el límite" es cuando σ(z) = 0.5, lo que equivale a z = 0. Esa ecuación w₁·x₁ + w₂·x₂ + b = 0 describe una línea recta en 2D (o un plano en 3D), y es precisamente la frontera de decisión: a un lado la neurona asigna una categoría, al otro lado, la otra.

🧩 Diagrama interactivo: 1 neurona con Sigmoid

Es el mismo esquema que en regresión lineal (suma ponderada z = w·x + b), pero ahora la función de activación es Sigmoid y la salida se interpreta como probabilidad de que la casa sea "cara".

En un modelo ya entrenado, es muy común que las entradas se hayan normalizado (re-escalado) antes de aprender los pesos. Aquí tú introduces los valores crudos (m², habitaciones, etc.) y la neurona los transforma internamente a una escala normalizada antes de combinarlos y pasar por la Sigmoid.

💡 Importante: en el modelo de regresión los parámetros \(w, b\) se entrenan para predecir directamente un precio (por ejemplo, miles de euros) a partir de las características. En cambio, en este modelo con Sigmoid los mismos tipos de entradas se usan para predecir la probabilidad de que la casa sea "cara" (etiqueta 1) frente a "barata" (etiqueta 0). Aquí los parámetros ya no significan "euros por m²", sino cuánto empuja cada característica a que \(z\) suba o baje, y por tanto a que \(\sigma(z)\) se acerque más a 1 (cara) o a 0 (barata).

  • La línea verde en el caso linealmente separable es la frontera de decisión del modelo: justo sobre la línea el valor interno es z = 0.
  • Un poco a la izquierda de la línea, el modelo tiene z < 0 y por tanto σ(z) < 0.5 → la neurona decide "barata" (clase 0).
  • Un poco a la derecha de la línea, el modelo tiene z > 0 y por tanto σ(z) > 0.5 → la neurona decide "cara" (clase 1).

🎯
¡Este es el punto clave! Para problemas NO linealmente separables (círculos, XOR, espirales), necesitamos capas ocultas con funciones de activación no lineales. Eso es lo que convierte a una neurona simple en una red neuronal poderosa.

📋 Código para copiar y pegar: frontera de decisión en 2D

Partiendo de los imports del Paso 0, este ejemplo genera un dataset linealmente separable en 2D, entrena una neurona con activación Sigmoid y dibuja la frontera de decisión donde z = 0, es decir, donde σ(z) = 0.5. 

import torch
    import torch.nn as nn
    import numpy as np
    import matplotlib.pyplot as plt

    np.random.seed(42)
    torch.manual_seed(42)

    # Dataset 2D linealmente separable
    n = 100
    x1 = np.random.uniform(-1, 1, n)
    x2 = np.random.uniform(-1, 1, n)
    X = np.stack([x1, x2], axis=1).astype(np.float32)

    # Frontera "real": w1 * x1 + w2 * x2 + b = 0
    w_true = np.array([2.0, -1.0], dtype=np.float32)
    b_true = 0.0
    z_true = X @ w_true + b_true
    y = (z_true >= 0).astype(np.float32).reshape(-1, 1)

    X_tensor = torch.from_numpy(X)
    y_tensor = torch.from_numpy(y)

    # Modelo logístico: 1 neurona con Sigmoid en 2D
    modelo = nn.Sequential(
      nn.Linear(2, 1),
      nn.Sigmoid()
    )

    criterio = nn.BCELoss()
    opt = torch.optim.SGD(modelo.parameters(), lr=0.1)

    for epoca in range(1000):
      y_pred = modelo(X_tensor)
      loss = criterio(y_pred, y_tensor)

      opt.zero_grad()
      loss.backward()
      opt.step()

    print(f"Loss final: {loss.item():.4f}")

    # Visualizar datos y frontera aprendida (z = 0)
    with torch.no_grad():
      w = modelo[0].weight[0].detach().numpy()
      b = modelo[0].bias[0].item()

    x1_grid = np.linspace(-1.2, 1.2, 100)
    if abs(w[1]) < 1e-6:
      x2_grid = np.zeros_like(x1_grid)
    else:
      x2_grid = -(w[0] * x1_grid + b) / w[1]

    plt.figure(figsize=(4, 4))
    plt.scatter(X[:, 0], X[:, 1], c=y_tensor.squeeze().numpy(), cmap="coolwarm", alpha=0.7)
    plt.plot(x1_grid, x2_grid, "k--", label="frontera z=0")
    plt.axhline(0, color="gray", linewidth=0.5)
    plt.axvline(0, color="gray", linewidth=0.5)
    plt.xlabel("x1")
    plt.ylabel("x2")
    plt.legend()
    plt.title("Frontera de decisión aprendida (σ(z)=0.5)")
    plt.show()

¿Qué es una Capa Oculta?

El ingrediente secreto de las redes neuronales

✅ Al terminar esta sección deberías poder:

  • Distinguir entre "más neuronas" y "más capas" y su efecto en la capacidad del modelo.

Ya vimos que una neurona sola solo puede dibujar líneas rectas. Para resolver problemas como XOR o círculos, necesitamos capas ocultas. Pero, ¿qué son exactamente?

🏭
Analogía: La fábrica de procesamiento

Imagina que quieres decidir si una foto muestra un gato. El proceso tiene etapas:

  • Entrada: Píxeles crudos (números)
  • Capa Oculta 1: Detecta bordes y líneas básicas
  • Capa Oculta 2: Combina bordes → detecta ojos, orejas, bigotes
  • Capa Oculta 3: Combina partes → detecta cara de gato
  • Salida: ¿Es gato? Sí/No

Cada capa oculta extrae representaciones más abstractas. ¡Por eso se llama "deep learning" — aprendizaje profundo!

🔗 Anatomía de una Red con Capas Ocultas

¿Por qué se llaman "ocultas"?

Porque no las vemos directamente. Solo vemos la entrada (datos) y la salida (predicción). Las capas intermedias "procesan" internamente — sus valores son representaciones aprendidas automáticamente.

🧮 ¿Qué hace cada neurona oculta?

Cada neurona calcula: σ(w₁x₁ + w₂x₂ + b). Con diferentes pesos, cada neurona "busca" un patrón diferente en los datos.

🔗 ¿Por qué combinarlas?

La siguiente capa combina los patrones detectados. Así, patrones simples (líneas) forman patrones complejos (círculos, formas).

📊 ¿Cuántas capas y neuronas?

Más neuronas: más patrones detectables
Más capas: patrones más abstractos

import torch
    import torch.nn as nn

    # Datos en círculo (clasificación binaria)
    def make_circle(n_puntos=400, radio=0.8):
      X = torch.empty(n_puntos, 2).uniform_(-1.2, 1.2)
      radios = X.pow(2).sum(dim=1).sqrt()
      y = (radios < radio).float().unsqueeze(1)
      return X, y

    X, y = make_circle()
    print("Proporción clase 1 (interior del círculo):", y.mean().item())

    # Modelo lineal: 2 -> 1 con Sigmoid (frontera recta)
    modelo_lineal = nn.Sequential(
      nn.Linear(2, 1),
      nn.Sigmoid()
    )

    criterio = nn.BCELoss()
    opt_lineal = torch.optim.SGD(modelo_lineal.parameters(), lr=0.5)

    for epoca in range(1000):
      y_pred = modelo_lineal(X)
      loss = criterio(y_pred, y)

      opt_lineal.zero_grad()
      loss.backward()
      opt_lineal.step()

      if epoca % 200 == 0:
        print(f"[Lineal] Epoch {epoca} | Loss = {loss.item():.4f}")

    with torch.no_grad():
      probs_lin = modelo_lineal(X)
      preds_lin = (probs_lin >= 0.5).float()
      acc_lin = (preds_lin == y).float().mean().item()
    print(f"Exactitud modelo lineal: {acc_lin:.3f}")

    # Modelo con capa oculta: 2 -> 8 -> 1 (frontera curva)
    modelo_profundo = nn.Sequential(
      nn.Linear(2, 8),
      nn.ReLU(),
      nn.Linear(8, 1),
      nn.Sigmoid()
    )

    opt_profundo = torch.optim.SGD(modelo_profundo.parameters(), lr=0.5)

    for epoca in range(1000):
      y_pred = modelo_profundo(X)
      loss = criterio(y_pred, y)

      opt_profundo.zero_grad()
      loss.backward()
      opt_profundo.step()

      if epoca % 200 == 0:
        print(f"[Oculta] Epoch {epoca} | Loss = {loss.item():.4f}")

      probs_deep = modelo_profundo(X)
      preds_deep = (probs_deep >= 0.5).float()
      acc_deep = (preds_deep == y).float().mean().item()
    print(f"Exactitud modelo con capa oculta: {acc_deep:.3f}")

📊 ¿Qué son los Logits?

Los logits son las salidas crudas de la última capa lineal de una red neuronal, antes de aplicar Sigmoid o Softmax.

🔢 Características
  • Rango: −∞ a +∞
  • No son probabilidades
  • Valor alto = más "confianza" en esa clase
🔄 Transformación
  • Binario: Sigmoid(logit) → prob
  • Multiclase: Softmax(logits) → probs
  • Resultado: valores entre 0 y 1
# Ejemplo:
logits = [2.0, 0.5, -1.0] ← salida de nn.Linear
probs = softmax(logits) = [0.71, 0.16, 0.04] ← ahora sí son probabilidades
🎯 ¿Y para multiclase? Softmax en la última capa

Ahora que entiendes qué son los logits, veamos cómo manejar más de 2 clases:

  • Softmax convierte logits en probabilidades por clase y garantiza Σ probs = 1.
  • Úsala cuando hay >2 clases (ej. gato, perro, pájaro); para binario, deja Sigmoid.
  • En código: salida logits → nn.Softmax(dim=1) o CrossEntropyLoss (que ya aplica softmax internamente).
📐 Fórmula de Softmax
σ(zi) = ezi
Σj ezj
✅ Propiedades
  • Cada salida ∈ (0, 1)
  • La suma de todas = 1
  • Preserva el orden de los logits
⚠️ Cuidado con la estabilidad
  • Exponenciales pueden explotar (overflow)
  • Solución: restar max(z) antes de aplicar exp
  • PyTorch/TensorFlow ya lo hacen internamente
💡 Nota sobre Loss Functions: En clasificación multiclase, la función de pérdida más común es Cross-Entropy Loss (que mide qué tan diferentes son las probabilidades predichas vs las reales). En PyTorch, nn.CrossEntropyLoss espera logits crudos (sin softmax) porque aplica log-softmax internamente de forma estable. (Veremos más sobre funciones de pérdida en el Apéndice)

def make_3class_data(N=300):

    W_true = torch.tensor([
        [2.0, -1.0],
        [-1.5, 2.0],
        [-1.0, -1.0],
    ])  # shape: [3, 2]
    b_true = torch.tensor([0.5, -0.5, 0.0])  # [3]
    logits_true = X @ W_true.T + b_true  # [N, 3]
    y = logits_true.argmax(dim=1)        # labels 0, 1, 2
    return X, y

print(torch.stack([y.bincount(), torch.arange(y.bincount().numel())]))

criterio = nn.CrossEntropyLoss()  # espera logits crudos y labels enteros
opt = torch.optim.SGD(modelo.parameters(), lr=0.1)

for epoca in range(200):

    loss = criterio(logits, y)  # softmax + log-softmax interno

    opt.zero_grad()
    loss.backward()
    opt.step()

Funciones de Activación: A Fondo

Softmax, ReLU, saturación y el problema del vanishing gradient

Las funciones de activación son el corazón de las redes neuronales. Sin ellas, no importa cuántas capas apiles: el resultado sería equivalente a una única transformación lineal. En esta sección exploraremos en profundidad cada función, sus derivadas, y por qué algunas causan problemas durante el entrenamiento.

🔴 Sigmoid (Función Logística)
σ(x) = 1
1 + e−x
Derivada: σ'(x) = σ(x) · (1 − σ(x))

Características:

  • Comprime cualquier valor a (0, 1)
  • Ideal para probabilidades binarias
  • Suave y diferenciable en todo punto
  • Centrada en 0.5, no en 0
⚠️ El Problema de Saturación

Cuando |x| es grande, σ(x) se aplana:

σ(5) = 0.9933 → σ'(5) = 0.0066
σ(10) = 0.99995 → σ'(10) = 0.00005

Resultado: Los gradientes se vuelven minúsculos → las capas profundas apenas aprenden.

📉 Visualización de la Derivada de Sigmoid
← Zona saturada (gradiente ≈ 0) Zona activa Zona saturada (gradiente ≈ 0) →
🔵 Tanh (Tangente Hiperbólica)
tanh(x) = ex − e−x
ex + e−x = 2σ(2x) − 1
Derivada: tanh'(x) = 1 − tanh²(x)

Ventajas sobre Sigmoid:

  • Salida centrada en 0 → mejor para capas ocultas
  • Derivada máxima = 1.0 (vs 0.25 de sigmoid)
  • Rango simétrico (-1, 1)
🔄 Sigmoid vs Tanh

Tanh es simplemente una versión reescalada y desplazada de Sigmoid:

tanh(x) = 2 · σ(2x) − 1

# En Python:
tanh = 2 * torch.sigmoid(2*x) - 1

Aún sufre saturación, pero menos severa que sigmoid.

🟢 ReLU (Rectified Linear Unit)
ReLU(x) = max(0, x)
Derivada: ReLU'(x) = 0 si x < 0, sino 1

¿Por qué revolucionó deep learning?

  • Derivada = 1 en zona positiva → sin saturación
  • Computacionalmente muy eficiente
  • Induce sparsity (muchas neuronas en 0)
  • Permite entrenar redes muy profundas
💀 El Problema de "Dead ReLU"

Si una neurona cae en zona negativa y se queda ahí:

Para x < 0: ReLU(x) = 0
Derivada = 0 → Gradiente = 0
→ ¡La neurona nunca se actualiza!

Solución: Leaky ReLU, PReLU, ELU

⚡ Por qué ReLU mitiga el Vanishing Gradient
❌ Con Sigmoid (5 capas)
∂L/∂a₅ × 0.25 × 0.25 × 0.25 × 0.25
Gradiente final: 0.25⁴ = 0.004 (¡casi nada!)
✓ Con ReLU (5 capas)
∂L/∂a₅ × 1 × 1 × 1 × 1
Gradiente final: 1⁴ = 1 (¡se preserva!)

MSE - Mean Squared Error

La función de pérdida estándar para regresión (predecir valores continuos)

¿Qué mide MSE? El promedio de los errores al cuadrado entre tus predicciones y los valores reales. Mientras más pequeño el MSE, mejor está prediciendo tu modelo. Un MSE de 0 significa predicciones perfectas.

yi
valor real (ground truth)
ŷi
predicción del modelo

📏 ¿Cuándo es "bueno" o "malo" el MSE?

El valor de MSE depende de la escala de tus datos. No hay un número mágico que sea "bueno" para todos los problemas.

✗ Podría ser muy malo

✓ Generalmente bueno

1. Hace positivos todos los errores

¡Parece perfecto pero no lo es!
✓ Con cuadrado

Refleja el error real
2. Penaliza más los errores grandes
Esto es deseable: un error de 4 es mucho peor que dos errores de 2.

Cross-Entropy Loss

La función de pérdida estándar para clasificación (categorías)

¿Qué mide Cross-Entropy? Qué tan "sorprendida" estaría la red si le dijeras la respuesta correcta. Si la red predice 90% de probabilidad para la clase correcta, la sorpresa (loss) es baja. Si predice solo 10%, la sorpresa es alta → loss alto.

BCEWithLogitsLoss

Para clasificación binaria - más estable numéricamente

BCE = −[y·log(σ(z)) + (1−y)·log(1−σ(z))]
donde z = logit (salida sin sigmoid), σ = sigmoid, y ∈ {0, 1}
✓ ¿Por qué BCEWithLogits?
  • Usa log-sum-exp trick para estabilidad
  • Evita underflow/overflow de sigmoid
  • Combina sigmoid + BCE en una operación
  • Gradientes más precisos
📊 Ejemplos numéricos
z=2.0, y=1: loss = 0.13 (bien)
z=-2.0, y=1: loss = 2.13 (mal)
z=0.0, y=1: loss = 0.69 (incertidumbre)
⚠️ ¿Qué son Underflow y Overflow?

Las computadoras tienen precisión limitada para representar números decimales. Cuando los números se vuelven extremadamente pequeños o grandes, ocurren problemas:

📉 UNDERFLOW
Número demasiado pequeño → se redondea a 0
sigmoid(-100) = ?
= 0.0000000...0000
≈ 0 (¡perdemos precisión!)
Problema: log(0) = -∞ 💥
📈 OVERFLOW
Número demasiado grande → se vuelve infinito
exp(1000) = ?
= 10434
≈ ∞ (¡número imposible!)
Problema: operaciones con ∞ = NaN 💥
🔍 El problema con sigmoid + log:
Paso 1
Logit muy negativo: z = -100
Paso 2
Sigmoid: σ(-100) = 1/(1+e100) ≈ 0.0 (underflow!)
Paso 3
Log: log(0.0) = -∞ 💥 ¡Error!
✓ BCEWithLogits evita esto:
Usa trucos matemáticos (log-sum-exp) para calcular log(sigmoid(z)) de forma estable, sin calcular sigmoid primero. Así evita que números muy pequeños o grandes rompan el cálculo.
💡 Analogía: Es como una calculadora de bolsillo: si el resultado tiene 200 dígitos, solo muestra "ERROR". BCEWithLogits es como una calculadora científica que sabe manejar números gigantes sin explotar.

Backpropagation ("propagación hacia atrás") es el algoritmo que calcula cómo debe cambiar cada peso para reducir el error. Usa la regla de la cadena del cálculo para propagar el gradiente desde la salida hasta la entrada.

🧭 ¿Qué es un Gradiente? (Intuición Visual)

El gradiente de una función te dice en qué dirección y qué tan rápido la función crece más rápidamente.

📐 Matemáticamente:
∇L = [∂L/∂w₁, ∂L/∂w₂, ...]

Vector de derivadas parciales respecto a cada peso

∂L/∂w > 0
↑w → ↑Loss
Debemos bajar w
∂L/∂w < 0
↑w → ↓Loss
Debemos subir w

El gradiente apunta "cuesta arriba" →
Nosotros vamos en dirección opuesta

🏔️ Analogía: Descenso de Gradiente = Bajar una Montaña

1. Tocas el suelo alrededor
→ Calculas el gradiente
2. Sientes la pendiente
→ ¿Qué dirección baja más?
3. Das un paso en esa dirección
→ Actualizas los pesos
4. Repites hasta llegar abajo
→ Convergencia (loss mínimo)

⚡ Forward Pass vs Backward Pass

➡️ FORWARD PASS
  • Propaga los datos hacia adelante
  • Calcula la predicción (ŷ)
  • Computa el loss
  • Guarda valores intermedios para backprop
output = model(x)
loss = criterion(output, y)
⬅️ BACKWARD PASS
  • Propaga el error hacia atrás
  • Calcula ∂L/∂w para cada peso
  • Usa la regla de la cadena
  • Acumula gradientes en .grad
loss.backward()
# model.layer.weight.grad tiene ∂L/∂w

⛓️ La Regla de la Cadena: El Corazón de Backprop

🤔
El gran problema: ¿Cómo sabe la red cuánto cambiar cada peso?
Tenemos miles de pesos. El error está al final de la red.
¿Cómo "culpamos" a cada peso por su parte del error?
📞 Analogía: El Juego del Teléfono
Imagina el juego del teléfono: un mensaje pasa de persona a persona y al final llega distorsionado. ¿Cuánto "culpa" tiene cada persona del error final?
➡️ Forward Pass
El mensaje (datos) viaja de izquierda a derecha, transformándose en cada paso.
⬅️ Backward Pass
La "culpa" (gradiente) viaja de derecha a izquierda, multiplicándose en cada paso.
🔗 La Regla de la Cadena en Una Frase
∂L/∂w = ∂L/∂a × ∂a/∂z × ∂z/∂w
"El gradiente del todo = producto de gradientes locales"
∂L/∂a
"¿Cuánto afecta a al error?"
∂a/∂z
"¿Cuánto afecta z a a?"
∂z/∂w
"¿Cuánto afecta w a z?"
💡 Intuición clave:
Es como preguntar: "Si cambio el peso w un poquito, ¿cuánto cambia el error L?"
La respuesta se obtiene multiplicando los efectos intermedios.
📚 ¿Nuevo en derivadas? Haz clic aquí para un repaso rápido

Una derivada mide cuánto cambia una cosa cuando cambias otra.

Ejemplo cotidiano:
Si conduces a 60 km/h, tu posición cambia 60 km por cada hora.
→ Velocidad = derivada de posición
En redes neuronales:
Si el peso sube 0.1, ¿cuánto sube el loss?
→ ∂L/∂w = derivada del loss

∂L
∂w

2️⃣ ∂L/∂w₂ (regla de la cadena)
ŷ = w₂·a₁ + b₂
∂(w₂·a₁)/∂w₂ = a₁,   ∂(b₂)/∂w₂ = 0
→ ∂ŷ/∂w₂ = a₁ + 0 = 0.5
∂L/∂w₂ = ∂L/∂ŷ × ∂ŷ/∂w₂ = 1.6 × 0.5 = 0.8
3️⃣ ∂L/∂a₁ (propagando hacia atrás)
ŷ = w₂·a₁ + b₂
∂(w₂·a₁)/∂a₁ = w₂,   ∂(b₂)/∂a₁ = 0
→ ∂ŷ/∂a₁ = w₂ + 0 = 2.0
∂L/∂a₁ = ∂L/∂ŷ × ∂ŷ/∂a₁ = 1.6 × 2.0 = 3.2
4️⃣ ∂L/∂z₁ (a través de ReLU)
a₁ = ReLU(z₁) → ReLU'(z₁) = 1 (porque z₁=0.5>0)
∂L/∂z₁ = ∂L/∂a₁ × ReLU'(z₁) = 3.2 × 1 = 3.2
5️⃣ ∂L/∂w₁ (regla de la cadena final)
z₁ = w₁·x + b₁ → ∂z₁/∂w₁ = x = 1.0
∂L/∂w₁ = ∂L/∂z₁ × ∂z₁/∂w₁ = 3.2 × 1.0 = 3.2
🔄 ACTUALIZACIÓN DE PESOS
¿Y ahora qué hacemos con los gradientes?
El gradiente ∂L/∂wi nos dice "si aumento wi, ¿cuánto sube el error?"
→ Si el gradiente es positivo: aumentar wi aumenta el error → debemos reducir wi
→ Si el gradiente es negativo: aumentar wi reduce el error → debemos aumentar wi
Por eso restamos el gradiente: vamos en dirección contraria al error.
Fórmula del Gradiente Descendente:
winuevo = wiviejo − η × ∂L/∂wi
η (eta) = learning rate = 0.1 → controla qué tan grandes son los pasos (muy grande = inestable, muy pequeño = lento)
w₂ (usamos ∂L/∂w₂ = 0.8)
2.0 − 0.1 × 0.8 = 2.0 − 0.08 = 1.92
w₁ (usamos ∂L/∂w₁ = 3.2)
0.5 − 0.1 × 3.2 = 0.5 − 0.32 = 0.18
b₂ (usamos ∂L/∂b₂ = 1.6)
0 − 0.1 × 1.6 = 0 − 0.16 = −0.16
b₁ (usamos ∂L/∂b₁ = 3.2)
0 − 0.1 × 3.2 = 0 − 0.32 = −0.32
💡 Observa: w₁ cambia más que w₂ (gradiente 3.2 vs 0.8). El error se "amplifica" al propagarse hacia atrás por la regla de la cadena.
⚠️ ¿Y si usáramos Sigmoid en vez de ReLU?
En el paso 4️⃣ multiplicamos por ReLU'(0.5) = 1. ¿Qué pasaría con otras activaciones?
✅ ReLU
Derivada = 1 (si z > 0)
El gradiente pasa intacto
3.2 × 1 = 3.2 ✓
⚠️ Sigmoid
Derivada máxima = 0.25
El gradiente se reduce a ¼
3.2 × 0.25 = 0.8 ↓
⚠️ Tanh
Derivada máxima = 1
Pero cerca de 0, típicamente ~0.4
3.2 × 0.4 = 1.28 ↓
🔴 El Problema del Desvanecimiento del Gradiente
Imagina una red con 10 capas usando Sigmoid. En cada capa multiplicamos por ~0.25:
Gradiente final = gradiente inicial × 0.2510 = gradiente × 0.00000095
¡El gradiente prácticamente desaparece! Las primeras capas casi no aprenden.
✅ Por eso usamos ReLU (y variantes)
ReLU: Derivada = 1 para z > 0. Sin desvanecimiento.
Leaky ReLU: Derivada = 0.01 para z ≤ 0. Evita "neuronas muertas".
GELU/SiLU: Suaves, usados en transformers modernos.
💀 Neuronas Muertas (Dying ReLU)
Si z ≤ 0, entonces ReLU'(z) = 0. El gradiente se bloquea completamente. Si una neurona "muere" (siempre produce valores negativos), nunca se recupera. → Por eso existen Leaky ReLU, PReLU, ELU...

🔥 PyTorch Autograd: ¡Todo Automático!

En la práctica, nunca calculas gradientes a mano. PyTorch construye un grafo computacional durante el forward pass y lo usa para calcular todos los gradientes automáticamente.

📊 ¿Qué es el Grafo Computacional?

Imagina que cada operación matemática que haces es un nodo en un diagrama. PyTorch va "grabando" todas las operaciones mientras calculas el forward pass, creando un grafo dirigido que conecta:

📥 Tensores de Entrada
x, pesos, biases
⚙️ Operaciones
multiplicación, suma, ReLU, etc.
📤 Salida Final
loss (el valor que queremos minimizar)
🎮 Visualización del Grafo (haz clic en "Construir Grafo" para animarlo)
El grafo muestra cómo PyTorch conecta operaciones para calcular gradientes
💡 ¿Por qué es tan útil?
  • No calculas derivadas: PyTorch conoce la derivada de cada operación básica (suma, producto, ReLU, etc.)
  • Regla de la cadena automática: El grafo permite aplicar la regla de la cadena "caminando hacia atrás"
  • Redes arbitrarias: Funciona para cualquier arquitectura, sin importar cuán compleja sea
  • Eficiente: Solo calcula los gradientes que necesitas (los que tienen requires_grad=True)
💻 Código PyTorch Completo
# 1. Definir tensores CON tracking

# 3. Backward (UNA línea!)

Backpropagation ("propagación hacia atrás") es el algoritmo que calcula cómo debe cambiar cada peso para reducir el error. Usa la regla de la cadena del cálculo para propagar el gradiente desde la salida hasta la entrada.

📋 Código para copiar y pegar: autograd simple

Este mini-ejemplo muestra cómo PyTorch calcula automáticamente los gradientes de una función sencilla y actualiza un peso siguiendo la dirección de descenso de gradiente.

import torch

# Datos sencillos: x y y_true fijos
x = torch.tensor([2.0])
y_true = torch.tensor([4.0])

# Peso con gradiente
w = torch.tensor([1.0], requires_grad=True)

for step in range(5):
    # Forward: y = w * x
    y_pred = w * x
    loss = (y_pred - y_true).pow(2).mean()  # MSE

    # Backward
    loss.backward()

    print(f"Paso {step} | w = {w.item():.3f} | grad = {w.grad.item():.3f} | loss = {loss.item():.3f}")

    # Actualizar peso manualmente (SGD simple)
    with torch.no_grad():
        w -= 0.1 * w.grad

    # Importante: limpiar gradiente para el siguiente paso
    w.grad.zero_()

b_true = torch.tensor([0.5])

logits_true = X_train @ w_true + b_true

print("Proporción clase 1:", y_train.mean().item())

# Modelo: neurona logística 2 -> 1
modelo = nn.Sequential(
    nn.Linear(2, 1),
    nn.Sigmoid()
)

criterio = nn.BCELoss()
opt = torch.optim.SGD(modelo.parameters(), lr=0.1)

for epoca in range(200):
    # 1. Forward
    y_pred = modelo(X_train)
    loss = criterio(y_pred, y_train)

    # 2. Backward
    opt.zero_grad()
    loss.backward()
    opt.step()

    if epoca % 20 == 0:
        print(f"Epoch {epoca} | Loss = {loss.item():.4f}")

with torch.no_grad():
    probs = modelo(X_train)
    preds = (probs >= 0.5).float()
    acc = (preds == y_train).float().mean().item()

print(f"Exactitud final: {acc:.3f}")
for epoch in range(100):
    optimizer.zero_grad()          # 1. Limpiar gradientes
    predictions = model(X_train)   # 2. Forward pass
