De que trata este capitulo?

Este capitulo es el puente entre la teoria y la practica. Aprenderas a crear graph embeddings (representaciones numericas de grafos) usando dos metodos fundamentales: Node2Vec (N2V) y GNNs, y los compararas en un problema real de clasificacion semi-supervisada.

Enfoque AML/Banca de esta guia

Los embeddings son la piedra angular de cualquier sistema GNN para AML. Sin embeddings de calidad, tu modelo no puede distinguir entre una cuenta legitima y una mula financiera. En este capitulo adaptamos los conceptos a:

  1. Embeddings transaccionales: Convertir patrones de transacciones en vectores numericos
  2. Deteccion de comunidades: Identificar clusters de cuentas sospechosas
  3. Semi-supervised learning: Detectar fraude cuando solo tienes pocas cuentas etiquetadas como sospechosas
  4. Transductivo vs Inductivo: Elegir el metodo correcto segun si tu red financiera es estatica o dinamica

Al terminar este capitulo seras capaz de:

  • Crear embeddings con Node2Vec y entender sus parametros (p, q, walk_length)
  • Crear embeddings con una GNN (GCN) usando PyTorch Geometric
  • Visualizar embeddings con UMAP para descubrir clusters
  • Comparar N2V vs GNN en un problema de clasificacion semi-supervisada
  • Entender la diferencia entre aprendizaje transductivo e inductivo
  • Comprender message passing como el motor de las GNNs

Node2Vec (N2V) — Embeddings con Random Walks

🧠 Que es Node2Vec? (la idea, sin formulas)

Imagina que quieres describir a cada persona de una empresa sin preguntarle nada directamente. Solo puedes caminar por los pasillos y anotar con quien te topas. Si caminas 1,000 veces empezando desde la oficina de Ana y siempre terminas viendo a Carlos, Luisa y Pedro, entonces sabes que Ana esta "cerca" de ellos — probablemente trabajan juntos.

Node2Vec hace exactamente eso con un grafo: manda "caminantes" a pasear por las aristas, anota que nodos aparecen juntos, y convierte esas observaciones en una lista de numeros (el embedding) para cada nodo.

🚶 La analogia del turista

Piensa en Node2Vec como un turista que explora una ciudad sin mapa:

Lo que hace el turistaLo que hace Node2Vec
Sale de un punto (su hotel) Inicia una caminata desde un nodo del grafo
En cada esquina elige una calle al azar En cada nodo elige una arista al azar para moverse
Camina un rato y apunta los lugares que visito Recorre N pasos y registra los nodos que visito
Repite el paseo muchas veces desde cada hotel Repite la caminata muchas veces desde cada nodo
Los lugares que siempre aparecen juntos estan "cerca" Los nodos que co-aparecen frecuentemente tendran embeddings similares

💡 De donde sale la idea? Word2Vec → Node2Vec

La conexion con Word2Vec (NLP) — clic para expandir ▼

En NLP, Word2Vec descubrio algo genial: si tomas un libro y anotas que palabras aparecen cerca de cada palabra, puedes crear numeros que capturan significado. Por ejemplo:

"El gato se sento en la alfombra"
"El perro se sento en el sofa"

Como "gato" y "perro" aparecen en contextos parecidos (antes de "se sento"), sus vectores quedan cercanos en el espacio de embeddings.

Node2Vec hizo la misma trampa pero con grafos: en vez de "frases" usa "caminatas aleatorias". Cada caminata es como una "oracion" donde los "nodos" son las "palabras":

// Word2Vec (texto):
Frase 1:   "el" "gato" "duerme" "en" "la" "cama"
Frase 2:   "el" "perro" "duerme" "en" "el" "sofa"

// Node2Vec (grafo):
Walk 1:   Libro_A → Libro_B → Libro_D → Libro_A → Libro_C
Walk 2:   Libro_A → Libro_C → Libro_B → Libro_D → Libro_E

Luego le das esas "oraciones de nodos" al mismo algoritmo que Word2Vec, y ¡listo! — tienes embeddings para cada nodo del grafo.

🍯 Analogia AML: Detectar lavado con "paseos" por la red

Imagina una red de transacciones bancarias

Cada cuenta es un nodo. Cada transferencia es una arista. Ahora mandas un "caminante" a pasear:

  • 🚶 Sale de la Cuenta A (sospechosa)
  • → Sigue una transferencia hasta Cuenta B (shell company)
  • → Sigue otra hasta Cuenta C (otra shell)
  • → Llega a Cuenta D (banco offshore)
  • → Regresa a Cuenta B...

Despues de miles de paseos, las cuentas que siempre aparecen juntas en los mismos circuitos tendran embeddings parecidos. Un modelo de ML puede detectar que ese "cluster" se parece a patrones conocidos de lavado — sin que nadie le haya ensenado las reglas explicitamente.

🤔 Pero espera... como se convierten los paseos en numeros?

El truco de la red neuronal "shallow"

Despues de generar miles de caminatas, Node2Vec entrena una red neuronal muy simple (una sola capa oculta) con esta tarea:

"Dado un nodo, predice que nodos aparecen cerca de el en las caminatas"

La red aprende a hacer eso cada vez mejor. Y aqui viene la magia: los pesos de la capa oculta SON los embeddings. No nos importa la prediccion final — nos importa lo que la red aprendio internamente para poder predecir.

ComponenteQue haceAnalogia
Capa de entrada Recibe un nodo (one-hot encoded) Le dices "este es el Libro A"
Capa oculta 🌟 Aprende una representacion comprimida El "resumen" del libro en N numeros — ESTE es el embedding
Capa de salida Predice vecinos probables "Este libro se compra junto con B, C y D"

🎯 Resumen antes de los parametros

Node2Vec no sabe nada de politica. Solo sabe que ciertos libros se compran juntos (las aristas). Pero al pasear miles de veces por la red de co-compras y entrenar una red neuronal, descubre solo que hay 3 grupos (izquierda, derecha, neutral) — porque los libros de cada grupo estan mas conectados entre si.

Ahora si, veamos los parametros que controlan como camina el turista...

⚙ Parametros Clave de N2V

Ahora que entiendes que hace Node2Vec, veamos los botones y perillas que controlan como lo hace. Cada parametro es como una instruccion que le das al "turista":

dimensions — Tamano del embedding ▼

Define cuantos numeros tendra el embedding de cada nodo.

🚶 Analogia del turista: Es como decidir cuantas preguntas le haces al turista sobre cada lugar. Si le haces 2 preguntas ("te gusto?" y "era caro?") tienes un perfil basico. Si le haces 64 preguntas, tienes un perfil muy detallado de cada lugar.

Valor estandar: 128 (pero en el libro usan 64 para comparar con GNN)

Mas dimensiones = perfil mas detallado pero mas costoso computacionalmente

Menos dimensiones = perfil mas compacto pero puede perder informacion

En AML: Para redes transaccionales complejas, 64-128 suele ser un buen rango

walk_length — Longitud de cada caminata ▼

Cuantos pasos da cada caminata aleatoria.

🚶 Analogia del turista: Es cuantas cuadras camina el turista antes de parar. Si camina 5 cuadras, solo conoce el barrio inmediato. Si camina 80 cuadras, cruza toda la ciudad y descubre conexiones lejanas.

Valor estandar: 80 pasos (en el libro usan 30)

Mas pasos = explora mas vecindario, captura contexto mas amplio

Menos pasos = solo captura vecindario inmediato

En AML: Caminatas largas pueden revelar cadenas de lavado que pasan por 5-10 intermediarios

num_walks — Numero de caminatas por nodo ▼

Cuantas veces repetir la caminata desde cada nodo.

🚶 Analogia del turista: Si el turista sale de su hotel una sola vez, tal vez tuvo mala suerte y tomo una ruta rara. Si sale 200 veces, los lugares que aparecen repetidamente son los que de verdad estan conectados con el hotel. Es como hacer una encuesta: mas respuestas = resultado mas confiable.

Valor estandar: 10 (en el libro usan 200 para mas precision)

Mas caminatas = representacion mas robusta pero mas lento

Menos caminatas = mas rapido pero mas ruidoso

p (return parameter) — Control de retroceso ▼

Controla la probabilidad de que la caminata regrese al nodo anterior.

🚶 Analogia del turista: Imagina que el turista llega a una plaza. ¿Puede darse la vuelta y regresar por donde vino? Con p alto, le dices "prohibido regresar, sigue adelante" (explora cosas nuevas). Con p bajo, le dices "puedes regresar, quédate por aqui" (profundiza en la misma zona).

p alto = menos probable que retroceda (explora nuevos caminos)

p bajo = mas probable que retroceda (se queda cerca del inicio)

p = 1 = equilibrado (valor por defecto)

q (in-out parameter) — BFS vs DFS ▼

Controla si la caminata explora cerca (BFS) o lejos (DFS).

🚶 Analogia del turista: Con q alto, el turista es como un vecino curioso: recorre todas las calles de su colonia antes de salir (BFS — conoce bien la zona). Con q bajo, es un aventurero: se va lo mas lejos posible, cruzando barrios enteros (DFS — descubre la estructura global).

En AML: q alto detecta comunidades locales (ej. una familia que mueve dinero entre sus cuentas). q bajo detecta cadenas largas (ej. dinero que viaja por 8 paises antes de llegar a su destino).

q > 1 = BFS (breadth-first) — explora vecinos cercanos, captura estructura local

q < 1 = DFS (depth-first) — explora nodos lejanos, captura estructura global

q = 1 = equilibrado (DeepWalk clasico)

BFS (flechas oscuras): explora cerca  |  DFS (flechas claras): explora lejos

Codigo Clave

import NetworkX as nx
from Node2Vec import Node2Vec

# 1. Cargar grafo
books_graph = nx.read_gml('polbooks.gml')

# 2. Inicializar N2V
node2vec = Node2Vec(books_graph,
    dimensions=64,
    walk_length=30,
    num_walks=200,
    workers=4)

# 3. Entrenar
model = node2vec.fit(window=10, min_count=1, batch_words=4)

# 4. Extraer embeddings
embeddings = {str(node): model.wv[str(node)]
    for node in books_graph.nodes()}

Visualizacion con UMAP

Por que UMAP y no visualizar directamente en 2D?

Puedes crear embeddings directamente en 2D (dimensions=2), pero UMAP produce mejor separacion de clusters porque:

  • Preserva estructura local Y global
  • Supera a t-SNE en mantener distancias globales
  • Permite crear embeddings ricos (64D) y luego reducirlos para visualizacion

Limitacion importante de N2V

N2V es transductivo: solo funciona con los nodos que vio durante el entrenamiento. Si llega un nodo nuevo al grafo, hay que reentrenar todo el modelo. Esto es critico en AML donde llegan cuentas nuevas todos los dias.

GNN Embeddings — Embeddings con Redes Neuronales de Grafos

La diferencia fundamental

Mientras N2V crea embeddings como paso separado (primero caminas, luego entrenas), las GNNs integran la creacion de embeddings directamente en el proceso de aprendizaje. Los embeddings se refinan mientras la red aprende la tarea.

🧠 Analogia: El Chisme de Vecindario

Imagina que vives en un vecindario y quieres saber que tipo de persona es cada vecino, pero solo puedes hablar con tus vecinos directos:

Lo que pasa en el vecindarioLo que hace la GNN
Ronda 1: Le preguntas a tus vecinos directos como son Capa 1: Cada nodo recibe los features de sus vecinos a 1 hop
Ronda 2: Tus vecinos te cuentan lo que sus vecinos les dijeron Capa 2: Ahora cada nodo tiene info de vecinos a 2 hops
Combinas todo lo que oiste en un resumen de tu zona Agrega (suma/promedia) los mensajes y los transforma = embedding

💡 La clave: Despues de 2 rondas de "chisme", cada nodo tiene un resumen que incluye info de su vecindario a 2 saltos de distancia — sin haber caminado por el grafo. La estructura llega a ti por difusion.

🎬 Animacion: Como la GNN crea un embedding

Observa como el nodo central recibe y agrega mensajes de sus vecinos en cada capa

Arquitectura: SimpleGNN

INPUT: Node Features (x) + Edge Index
Features iniciales de cada nodo + estructura del grafo
↓
GCNConv Layer 1 (Message Passing)
Cada nodo agrega informacion de sus vecinos
↓
ReLU + Dropout
Activacion no-lineal + regularizacion
↓
GCNConv Layer 2 (Message Passing)
Segunda ronda de agregacion de vecinos
↓
OUTPUT: Node Embeddings (64D)
Cada nodo tiene un vector de 64 numeros

Codigo de la Arquitectura

class SimpleGNN(torch.nn.Module):
    def __init__(self, num_features, hidden_channels):
        super().__init__()
        self.conv1 = GCNConv(num_features, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, hidden_channels)

    def forward(self, x, edge_index):
        x = self.conv1(x, edge_index)  # Message passing 1
        x = torch.relu(x)            # Activacion
        x = torch.dropout(x, p=0.5)   # Regularizacion
        x = self.conv2(x, edge_index)  # Message passing 2
        return x

Preparacion de Datos

Conversion: NetworkX → PyTorch Geometric ▼

PyG necesita los datos en formato tensor. La conversion es simple:

data = from_NetworkX(books_graph)

Esto traduce la lista de aristas y atributos de nodos/aristas a tensores de PyTorch.

Inicializacion de Features: Xavier Initialization ▼

Cuando no tienes features predefinidas para los nodos (como en el dataset de libros politicos), se inicializan aleatoriamente.

Xavier Initialization

En vez de numeros aleatorios simples, Xavier usa una distribucion que mantiene la variabilidad consistente entre capas. Esto evita dos problemas:

  • Vanishing gradients: Los gradientes se hacen tan pequenos que la red deja de aprender
  • Exploding gradients: Los gradientes se hacen tan grandes que la red se vuelve inestable

Alternativa: usar los embeddings de N2V como features iniciales (y obtener mejor rendimiento!).

Generacion de Embeddings

# Inicializar modelo
model = SimpleGNN(num_features=64, hidden_channels=64)

# Modo evaluacion (sin dropout, sin gradientes)
model.eval()
with torch.no_grad():
    gnn_embeddings = model(data.x, data.edge_index)

# Convertir a numpy para analisis
gnn_embeddings_np = gnn_embeddings.detach().cpu().numpy()

Dato importante

Incluso sin entrenar (solo un forward pass), la GNN ya produce embeddings utiles porque el message passing captura la estructura topologica del grafo. Con entrenamiento, los embeddings se optimizan para la tarea especifica.

Resultados en el Political Books Dataset

Adaptabilidad a Nuevos Grafos ▼

GNN: Aprende una funcion que mapea features a embeddings. Esta funcion se puede aplicar a nodos nuevos sin reentrenar.

N2V: Debe reentrenarse para cada nuevo grafo o conjunto de nodos.

Ejemplo AML: Si tu banco incorpora 1,000 cuentas nuevas cada dia, con GNN puedes generar sus embeddings instantaneamente. Con N2V, tendrias que reentrenar el modelo cada dia.

Integracion de Features ▼

GNN: Usa features de nodos (edad cuenta, patron depositos, pais) ademas de la estructura del grafo.

N2V: Solo usa la topologia (quien conecta con quien). Ignora features adicionales.

Optimizacion Especifica por Tarea ▼

GNN (end-to-end): Los embeddings se optimizan durante el entrenamiento para la tarea especifica (detectar fraude, clasificar cuentas). El embedding se ajusta a lo que necesitas.

N2V: Crea embeddings "genericos" que luego alimentan a otro modelo separado.

Cuando N2V es mejor que GNN?

  • Grafo estatico y bien definido que no cambia
  • Datos escasos o ruidosos (N2V funciona sin features de nodos)
  • Tarea exploratoria donde quieres ver la estructura rapidamente
  • Cuando necesitas algo rapido y simple como baseline

Aprendizaje Semi-Supervisado

Que es semi-supervised learning?

Entrenar un modelo con pocos datos etiquetados + muchos datos sin etiquetar. En el capitulo: solo el 20% de los libros tienen etiqueta politica visible, y el modelo debe predecir el 80% restante.

Por que es crucial para AML?

En la vida real, solo tienes unas pocas cuentas confirmadas como sospechosas (por investigaciones anteriores, SARs). Pero tienes millones de cuentas sin etiquetar. El aprendizaje semi-supervisado es perfecto para este escenario.

En general, los modelos GNN end-to-end superan al Random Forest, especialmente en F1 score (balance entre precision y recall).

El modelo de 4 capas supera al de 2 capas. Mas capas = mas hops de message passing = captura relaciones mas lejanas.

Representaciones y Embeddings

Representaciones basicas vs Embeddings ▼
RepresentacionDescripcionEjemplos
BasicaBuenas para algoritmos analiticos y recorridos de red. Info: vecinos de nodos/aristasLista de adyacencia, Edge list, Matriz de adyacencia
Transductiva (shallow)No sirven para datos no entrenados. Dificiles de escalarDeepWalk, N2V, TransE, RESCAL
InductivaSe generalizan a grafos nuevos y estructuralmente diferentes. Vectores en espacio continuoGNNs, Transformers

Transductivo vs Inductivo

Aprendizaje Transductivo (Shallow) ▼

Caracteristicas

  • Optimiza embeddings individualmente para cada nodo
  • Necesita todo el grafo disponible durante entrenamiento
  • No generaliza a nodos o grafos nuevos
  • Agregar nodos nuevos requiere reentrenar todo
  • Costoso computacionalmente para grafos grandes

Ejemplos: DeepWalk, N2V, matrix factorization

Aprendizaje Inductivo ▼

Caracteristicas

  • Aprende una funcion para generar embeddings
  • Puede generar embeddings para nodos nunca vistos
  • Usa features de nodos + estructura
  • Mas flexible y escalable para grafos dinamicos

Ejemplos: GraphSAGE, GCNs, GATs

Ejemplos Practicos: Cuando usar cada uno? ▼

Transductivo: Deteccion de comunidades en red social

Usa todo el grafo para identificar comunidades. El inductivo no aprovecharia toda la estructura porque solo procesa subconjuntos.

Inductivo: Deteccion de spam en email

Llegan emails nuevos constantemente. El modelo debe clasificar sin reentrenar. Transductivo seria impractico.

N2V: Random Walks en Profundidad

DeepWalk vs Node2Vec ▼

DeepWalk (2014): El pionero. Caminatas 100% aleatorias (sin sesgo). Todos los vecinos tienen la misma probabilidad.

Node2Vec (2016): Mejora DeepWalk con parametros p y q para controlar si exploras cerca (BFS) o lejos (DFS).

Para obtener DeepWalk con N2V: p=1, q=1 (sin sesgo).

Despues de las caminatas, una red neuronal shallow (1 hidden layer) aprende a predecir el siguiente nodo. El hidden layer se convierte en el embedding del nodo.

Message Passing como Deep Learning

Anatomia de una capa de Message Passing ▼
AGGREGATE
Recoger mensajes de vecinos a 1 hop
↓
TRANSFORM
Aplicar transformacion no-lineal (neural network)
↓
UPDATE
Actualizar el embedding del nodo

Numero de capas = Numero de hops

Cada capa de message passing recoge info de vecinos a 1 hop. Con 3 capas, cada nodo "ve" hasta 3 hops de distancia. El alcance crece exponencialmente (principio de "6 grados de separacion").

GNN vs Feed-Forward Neural Network ▼
AspectoMLP (Feed-Forward)GNN
InputVector de features (una fila)Grafo completo (nodos + aristas)
CapasLinear + ActivationMessage Passing + Activation
Info de vecinosNoSi (agregacion de vecinos)
Hidden layers sonRepresentaciones internasEmbeddings del grafo
OutputPrediccionEmbeddings + Prediccion

El truco elegante de las GNNs

En cada capa de message passing, se aplican muchas redes neuronales pequenas a nivel de cada nodo. Cada nodo tiene su propia transformacion local. Es como tener miles de mini-redes aprendiendo simultaneamente sobre datos locales.

Terminologia Clave

Shallow Methods

Metodos sin deep learning. Usan lookup tables en vez de modelos generalizados. Ej: N2V, DeepWalk, matrix factorization.

Encoder

Cualquier metodo que produce un embedding (representacion de baja dimension). Las GNNs son una clase de encoders, similar a Transformers.

End-to-End

Proceso completo en un solo modelo: datos crudos → embeddings → prediccion. Sin pasos separados. Las GNNs son end-to-end.

Embeddings en AML & Banca

Por que los embeddings son cruciales para AML?

Los sistemas AML tradicionales usan reglas fijas (ej: "alerta si transaccion > $10,000"). Los embeddings permiten capturar patrones complejos de comportamiento que las reglas no pueden detectar: cadenas de mulas, comunidades sospechosas, cuentas puente ocultas.

Embeddings Transaccionales para Mulas Financieras ▼

El escenario

Tienes una red de 50,000 cuentas bancarias con millones de transacciones. Solo 200 cuentas han sido confirmadas como mulas. Como detectas las demas?

Solucion con Embeddings

  1. Crear el grafo: Cuentas = nodos, Transacciones = aristas
  2. Features de nodo: Antiguedad, saldo promedio, # beneficiarios, velocidad de retiro
  3. Features de arista: Monto, hora, frecuencia, patron temporal
  4. Generar embeddings con GNN (inductivo, para cuentas nuevas)
  5. Semi-supervised: Usar las 200 mulas conocidas como etiquetas para clasificar las 49,800 restantes

Resultado esperado: Las cuentas mula tendran embeddings similares entre si y diferentes de las cuentas legitimas. Un clasificador puede separar ambos grupos.

Analogia: N2V como Fotografia, GNN como Video ▼

N2V = Fotografia

Captura un instante congelado de tu red financiera. Perfecto para analizar "como se veia la red el dia X". Pero si la red cambia, necesitas tomar otra foto.

GNN = Camara de Video

Se adapta en tiempo real. Puede procesar la red financiera a medida que evoluciona: nuevas cuentas, nuevas transacciones, nuevos patrones. No necesitas "tomar otra foto".

# pip install torch torch-geometric

import torch
from torch_geometric.nn import GCNConv
from torch_geometric.utils import from_networkx
import networkx as nx

# 1. Cargar y convertir
G = nx.read_gml('datos/polbooks.gml')
data = from_networkx(G)

# 2. Inicializar features (Xavier)
data.x = torch.empty((105, 64))
torch.nn.init.xavier_uniform_(data.x)

# 3. Definir SimpleGNN
class SimpleGNN(torch.nn.Module):
  def __init__(self, in_ch, hid_ch):
    super().__init__()
    self.conv1 = GCNConv(in_ch, hid_ch)
    self.conv2 = GCNConv(hid_ch, hid_ch)
  def forward(self, x, edge_index):
    x = self.conv1(x, edge_index)
    x = torch.relu(x)
    x = self.conv2(x, edge_index)
    return x

# 4. Generar embeddings (sin entrenar)
model = SimpleGNN(64, 64)
model.eval()
with torch.no_grad():
  emb = model(data.x, data.edge_index)
print(emb.shape) # torch.Size([105, 64])

Los 7 Puntos Mas Importantes

  1. Embeddings transforman grafos complejos en vectores numericos de baja dimension, preservando estructura y relaciones.
  2. Node2Vec (N2V) crea embeddings via random walks. Es transductivo: no generaliza a nodos nuevos.
  3. GNN embeddings son inductivos: generalizan a datos nuevos integrando features + topologia.
  4. Semi-supervised learning es ideal para AML: pocas cuentas etiquetadas, muchas sin etiquetar.
  5. N2V + GNN combinados dan el mejor rendimiento (89.29% F1) al usar N2V embeddings como features iniciales de la GNN.
  6. Message passing es el motor de las GNNs: cada capa agrega info de vecinos a 1 hop. Mas capas = mas alcance.
  7. Para AML en produccion, usar GNN (inductivo) para manejar cuentas nuevas. N2V para investigaciones puntuales.
